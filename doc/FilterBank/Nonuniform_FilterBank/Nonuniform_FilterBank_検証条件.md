# Nonuniform FilterBank 検証条件

## 1. 目的

本書は、`Nonuniform_FilterBank_設計方針.md` に対する
検証条件上の重要な前提を補足する。

特に、beamforming 接続および実用評価で用いる
受波器アレイ形状を明確化する。

---

## 2. 検証用アレイ形状

非均一フィルタバンクの検証で標準的に用いるアレイは、
以下の特徴を持つスパース直線アレイとする。

- 直線アレイである
- 中央部は高密度である
- 端部は低密度である
- すなわち、中央が密で端が疎な形状を持つ

概念的には、以下のような配列を想定する。

```text
sparse edge | dense center | sparse edge
```

これは、実運用で想定している

- 低域では広い開口を使う
- 高域では中央の高密度サブアレイを使う

という方針と整合する。

---

## 3. この形状を標準条件にする理由

均一 ULA だけで検証すると、

- 方式の良否
- アレイ設計の良否

が混ざりやすい。

一方、今回の不均一フィルタバンクは、
周波数帯域ごとに異なる分解能と異なる使用チャネル数を
前提とする。

したがって検証時のアレイも、

- 中央の dense region を高域で使う
- 外側の sparse region を低域で使う

という実用条件に合わせるべきである。

---

## 4. 検証時の扱い

beamforming 接続以降の主検証条件では、
以下を標準とする。

- 検証用アレイは、中央が密で端が疎なスパース直線アレイを用いる
- leaf band ごとに `used_channels` を切り替える
- 高域ほど中央寄りの高密度チャネル群を主に使う
- 低域ほど端側まで含めた広開口を使う

均一 ULA は補助比較には使用してよいが、
主たる評価条件にはしない。

---

## 4.1 初期代表条件

設計・検証を進めるための初期代表条件として、
まず以下を置く。

### アレイ

- `32 ch`
- dense center: `16 ch @ 0.01 m`
- sparse edge: 左右 `8 pair @ 0.04 m`

代表位置:

```text
[-0.395, -0.355, -0.315, -0.275, -0.235, -0.195, -0.155, -0.115,
 -0.075, -0.065, -0.055, -0.045, -0.035, -0.025, -0.015, -0.005,
  0.005,  0.015,  0.025,  0.035,  0.045,  0.055,  0.065,  0.075,
  0.115,  0.155,  0.195,  0.235,  0.275,  0.315,  0.355,  0.395] m
```

### leaf ごとの暫定使用チャネル数

| leaf band | used channels |
|---|---:|
| `0 - 128 Hz` | `32` |
| `128 - 256 Hz` | `32` |
| `256 - 512 Hz` | `24` |
| `512 - 1024 Hz` | `20` |
| `1024 - 2048 Hz` | `16` |
| `2048 - 4096 Hz` | `12` |
| `4096 - 8192 Hz` | `8` |
| `8192 - 16384 Hz` | `4` |

この条件では、
低域から高域へ向かって
中心対称な contiguous subset を縮める運用を基本代表とする。

これは最終固定値ではなく、

- dense center / sparse edge の実用条件
- 処理量見積りに使う代表条件
- 今後の beamforming 接続検証の出発点

として使う。

---

## 5. `Nonuniform_FilterBank_設計方針.md` との関係

`Nonuniform_FilterBank_設計方針.md` の内容はそのまま有効である。

本書は、その設計方針に対して

- 検証時に使うアレイ形状
- 実用評価時の標準条件

を追加で固定するための補足文書である。
