# 共分散積分3方式比較設計

## 1. 目的と範囲

本検討はMVDR重みの性能比較には進まず、MVDRへ入力する周波数別空間共分散行列をどのように推定するかだけを比較する。

3方式は最終出力を共通して次のshapeとする。

```text
R[ch_i, ch_j, k]
shape = [n_ch, n_ch, n_bin]
dtype = complex64
```

比較時は同一の入力信号、窓関数、FFT正規化、周波数軸、積分時間を使用する。MVDR重み、BL改善量、採否判定は本設計の責務に含めない。

## 2. 共通記号

| 記号 | 意味 | 単位・shape |
|---|---|---|
| `fs` | サンプリング周波数 | Hz |
| `delta_f` | 共分散分析幅 | Hz |
| `N` | FFT長 `fs / delta_f` | sample |
| `X_m,q[k]` | channel `m`、snapshot `q`、bin `k` のFFT値 | complex |
| `R_q[k]` | snapshot `q` の瞬時空間共分散 | `[n_ch,n_ch]` |
| `tau[m,theta]` | 基準点に対する到来遅延 | s |
| `d[m,theta]` | `tau*fs`から作る切り出し遅延 | sample |

瞬時共分散は全方式で次式に統一する。

```text
R_q[i,j,k] = X_i,q[k] * conj(X_j,q[k])
```

窓のエネルギー利得とFFT長によるscaleは方式間で同一にする。相関係数ではscaleが相殺されるが、共分散の対角powerと方式間誤差を比較するため省略しない。

## 2.1 全方式共通の指数積分規約

3方式の積分はすべて指数積分とする。`coef`は過去共分散に掛ける係数ではなく、今回観測した瞬時共分散に掛ける更新係数として定義する。

```text
R_next = (1 - coef) * R_previous + coef * R_instantaneous
0 < coef <= 1
```

`coef`が小さいほど過去を長く保持する。時間方向に独立snapshotが`rate [snapshot/s]`で到来し、等価積分時間を`T [s]`とする場合は、次を基準式とする。

```text
coef = min(2 / (1 + rate*T), 1)
```

この式では`rate*T`が等価独立snapshot数である。指数重み`coef*(1-coef)^q`の有効snapshot数は次式となり、上の定義と一致する。

```text
N_effective = (2 - coef) / coef
```

初回更新ではゼロ行列からの立ち上がりbiasを避けるため、共分散の加重和`S`と重み和`Z`を別々に更新し、`R=S/Z`とする正規化指数積分も比較する。

## 3. 方式1: 1 Hz長時間FFT共分散

`delta_f=1 Hz`では`N=fs` sampleを1 snapshotとしてFFTする。1秒ごとに各binの外積を作り、時間方向へ積分する。

```text
x_frame: [n_ch, fs]
X:       [n_ch, n_bin]
R_inst:  [n_ch, n_ch, n_bin]
R_out:   [n_ch, n_ch, n_bin]
```

1 frame、1 binから得られる`R_inst`は外積1個なのでrank 1以下である。したがって「1秒に1回積分」は、1秒frameを複数時刻にわたって指数積分する意味とする。1秒分だけでフルランク共分散が得られるわけではない。

非重複1秒frameでは`rate=1 snapshot/s`とする。

### 長所

- 1 Hz分解能を直接得られる。
- 1秒区間にわたる狭帯域定常信号ではbin leakageを抑えやすい。

### 短所

- FFT次数、待ち時間、作業領域が大きい。
- 1秒内で変化する信号を1 snapshotへ混合する。
- 同じ積分時間ではsnapshot数が少なく、共分散rankと追従性が低い。

## 4. 方式2: 短FFT block共分散積分

分析幅を`delta_f Hz`とすると、FFT長は`N=fs/delta_f` sample、1秒あたりの非重複block数は`delta_f`となる。

```text
x_block: [n_ch, N]
X_block: [n_ch, n_bin]
R_block: [n_ch, n_ch, n_bin]
R_out:   [n_ch, n_ch, n_bin]
```

block開始位置は`q*N` sampleとする。overlapを使用する場合はsnapshot数ではなく、有効独立snapshot数も記録する。

非重複blockでは`rate=fs/N=delta_f snapshot/s`とする。overlapを使用する場合、単純なblock到来数を`rate`にすると相関snapshotを独立と数えてしまうため、窓とhopから推定した有効rateも併記する。

### 長所

- 方式1より低遅延・低FFT次数である。
- 同一積分時間でsnapshot数を増やせる。
- 現行`ShortFFTCovarianceAccumulator`がこの方式に相当する。

### 短所

- 到来時間差がblock内で無補償のため、有限窓内に入る波形区間がchannel間でずれる。
- 到来時間差が信号の相関時間やtransient長に対して無視できない広帯域・非定常信号では、空間相関を過小評価する可能性がある。

## 5. 方式3: 方位別遅延整合snapshot選別共分散

## 5.1 遅延表

到来遅延`tau[m,theta]`から、切り出しに使う整数遅延と位相復元に使う残差遅延を分離する。

```text
d_int[m,theta] = round(tau[m,theta] * fs)
tau_residual[m,theta] = tau[m,theta] - d_int[m,theta] / fs
```

高周波では1 sample未満の誤差でも相関値が変わるため、整数遅延だけで方式3を評価して完了とはしない。

## 5.2 中心サンプルテーブル

方式3の中心サンプルテーブルを`channel_center[segment,ch,beam]`、
shape `[2,n_ch,n_beam]`、dtype `int32`とする。segment 0に0–90度の159中心からなる
長方形を作り、segment 1にはこの長方形をchannel方向へ上下反転した180–90度の
159中心を置く。各beamは1秒につき中心を1個だけ持つ。

```text
table_1[ch,beam] = table_0[n_ch-1-ch,beam]
```

中心から取得するsnapshot長はFFT次数`N=128`とする。方式2のように1秒を
`fs/N=256`個の非重複blockへ分割する処理ではない。159個の方位中心からそれぞれ
128 sampleを切り出し、異なる方位の切り出し区間が重なることを許可する。

```text
snapshot[ch,n,beam]
    = signal[ch,channel_center[ch,beam]-N//2+n]
shape = [n_ch,N,n_beam] = [n_ch,128,159]
```

block切り出しoffsetは`[-N//2, ..., N-N//2-1]`である。中心表の生成時に
全snapshotが1秒frame内へ収まるようscaleするが、中心間隔からblock数を導出しない。

中心テーブルはアレイ幾何、`fs`、音速、beam数、block長が変わらない限り不変である。初期化時に`CovarianceSnapshotCenterSchedule`を1回だけ作り、frame処理では保持済みscheduleの`extract_blocks()`を呼ぶ。毎frameの中心表再生成は禁止する。

## 5.2.1 方位一致テーブルと1秒切替

中心表と同じsegment軸を持つ方位一致テーブル`direction_match[segment,local_beam]`を保持する。全方位積分先は0–180度を317点で表し、90度だけを両segmentで共有する。

```text
segment 0: local 0..158 -> global 0..158       # 0→90度
segment 1: local 0..158 -> global 316..158     # 180→90度
```

処理秒counterが偶数ならsegment 0、奇数ならsegment 1の中心表と方位一致表を同時に選ぶ。選択した159方位だけを更新し、非選択方位の共分散は減衰させず前回値を保持する。異なる物理方位を同じ積分先へ混ぜてはいけない。

各local beamは選択された1秒で1個のsnapshotを持つ。方位一致表が示すglobal方位の
前回共分散と、今回の瞬時共分散を次式で1回更新する。

```text
R_next[theta] = (1-coef)*R_previous[theta]
                + coef*X[theta,q]X[theta,q]^H
```

これにより1秒だけを見た場合は選択された90度範囲だけを積分し、次の1秒では中心表と方位一致表を切り替えて反対側90度を積分する。

## 5.3 方位別snapshot

保持済み中心`center[segment,m,theta]`からsnapshotを切り出す。

```text
x_snapshot[m,n,theta]
    = x[m, center[segment,m,theta] - N//2 + n]
shape = [n_ch,N,n_beam]
      = [n_ch,128,159]
```

axis=0はchannel、axis=1はsnapshot内sample、axis=2はbeamである。

## 5.4 位相復元

式は実指数ではなく、虚数単位を含む複素位相回転でなければならない。

```text
phase_restore[m,k,theta] = exp(-j * 2*pi * f[k] * d_int[m,theta]/fs)
                              * exp(-j * 2*pi * f[k] * tau_residual[m,theta])
X_restored = X_shifted * phase_restore
```

符号は`x_shifted[n]=x[n+d]`のDFTが`exp(+j2*pi*f*d/fs)X(f)`になる規約に対する逆回転である。spflowの到来遅延・ステアリング符号と単体試験で一致させる。

重要なのは、位相復元しても有限窓が観測した時間区間の違いまでは元に戻らないことである。方式3の利点候補は「到来方向に対応する同一波面の有限時間区間を各channelから選ぶこと」であり、定常無限長信号では方式2と同値へ近づくべきである。

## 5.5 beam別共分散と相関値

位相復元後のsnapshotからbeam別共分散を時間方向へ指数積分する。この時間積分は方式1・2と同じ更新式を使い、各beamに対して独立な`S_theta`と`Z_theta`を保持する。

```text
R_theta[i,j,k,theta]
shape = [n_ch,n_ch,n_bin,n_beam]
```

正規化相関は次式とする。

```text
corr[i,j,k,theta] = abs(R[i,j,k,theta])
                      / sqrt(real(R[i,i,k,theta]) * real(R[j,j,k,theta]))
```

分母が0または数値下限未満の場合は相関0とする。対角を除く最大値を`rho_max[k,theta]`とする。

```text
rho_max[k,theta] = max(i != j) corr[i,j,k,theta]
w[k,theta] = rho_max[k,theta] > 0.6
```

`calculate_maximum_spatial_correlation_table`は全pair相関の4次元作業配列を生成せず、
上三角channel pairを逐次走査して`rho_max[theta,k]`を`float32`で返す。対角自己相関は
必ず除外し、`Rii*Rjj`が数値下限以下のpairは相関0とする。

最初の10秒評価は9ch対称ULA、128 sample snapshot、0–180度317方位、広帯域source
1–4 kHz、方位40度、SL 0 dB re input RMS、空間白色雑音NL -32 dB re input RMS/√Hz、
`coef=0.25`で行う。出力は`maximum_correlation[direction,bin]`、shape `[317,65]`とし、
imagesc相当のPNGとNPZを保存する。

`max`はchannel数が増えるほど偶然の高相関1組に反応しやすい。そのため0.6は採用済み閾値ではなく比較候補とし、同時に次も保存する。

- 非対角相関のmedian
- 非対角相関の95 percentile
- 最大固有値 / trace
- 閾値を超えたchannel pair比率

## 5.6 beam方向の指数積分

「同一方位方向の共分散を足す」は、各`theta`で時間snapshotを先に指数積分し、その後、同一binで有効beamをbeam方向へ指数積分する二段階と解釈する。

beam方向の更新係数を`coef_beam`とする。閾値選別を含むため、共分散加重和`S`と有効重み`Z`を別々に更新する。

```text
S_next[k] = (1-coef_beam)*S_previous[k]
              + coef_beam*w[k,theta]*R_theta[k,theta]
Z_next[k] = (1-coef_beam)*Z_previous[k]
              + coef_beam*w[k,theta]
R_out[k] = S_next[k] / Z_next[k]
```

`Z`が数値下限以下の場合は方式2の`R_out[k]`へfallbackする。集約後は数値誤差を除くため、`(R + R^H)/2`でHermitian化する。正定値性は保証されないため、固有値最小値とrankを観測するが、対角ローディングやMVDR重み評価は次工程とする。

## 5.7 `coef_beam`を先に式で確定しない理由

方式1・2の`rate`は時間当たりのsnapshot数として定義できる。一方、方式3のbeam方向積分では更新軸が時間ではなく角度であり、`rate*T`に対応する無次元量を一意に定義できない。

候補には少なくとも次がある。

- `rate_theta [beam/deg] * T_theta [deg]`
- 1回の走査で選択された有効beam数
- 相関のある隣接beamをまとめた有効独立beam数

また、指数積分は可換でないため、0度から180度へ走査する場合と逆順で`R_out`が変わる。これは時間EMAには自然な性質だが、角度集合の集約では方位方向の人工的な非対称性になる。

従って、`coef_beam`は算出式を先に固定せず、次の値を直接与えて共分散特性をシミュレーションする。

```text
coef_beam in {1.0, 0.5, 0.25, 0.125, 0.0625, 0.03125, 0.015625}
```

各係数で次の走査順を比較する。

- 方位昇順
- 方位降順
- broadsideから左右交互
- 固定seedのランダム順
- 昇順結果と降順結果の平均

係数ごとに`N_effective=(2-coef_beam)/coef_beam`、有効beam数、昇順・降順差、Frobenius誤差、trace、固有値、相関指標を保存する。走査順依存が許容範囲に収まり、広帯域・transient条件で方式2より真値誤差が改善する範囲を確認してから、`rate_theta`と`T_theta`の定義を確定する。

現行`DelayAlignedBeamCovarianceAccumulator`は全beam共分散を単純加算しており、方式3の完成形ではない。

### 6.1 方位ごとの更新率と忘却係数

方式3では、1秒区間ごとに90度分の159 beamだけを更新し、次の1秒区間では上下反転した中心サンプル表と方位一致表へ切り替える。従って、0度から180度までの全317方位のうち、90度以外の方位は2秒に1回、両区間で共有される90度だけは1秒に1回更新される。

時間積分定数を`T [s]`、ある方位が1秒間に更新される平均回数を`rate [update/s]`とすると、忘却係数は次式で方位ごとに求める。

```text
coef(theta) = 2 / (1 + rate(theta) * T)
```

例えば`T=10 s`では、通常方位は`rate=0.5 update/s`なので`coef=2/6`、90度は`rate=1 update/s`なので`coef=2/11`となる。この定義により、更新頻度が異なる90度だけが他方位より短い実時間で追従することを避ける。共分散更新式は全方式で次に統一する。

```text
R_next = (1 - coef(theta)) * R_previous + coef(theta) * (X X^H)
```

### 6.2 ベクトル化とメモリ制約

中心サンプルからの切り出し、FFT、瞬時共分散の計算はbeam chunkごとにNumPyで一括処理する。これによりbeam単位のPython loopを避けつつ、`[n_beam, n_ch, n_ch, n_bin]`の瞬時共分散全体を同時に保持しない。最大チャネル間相関も上三角のチャネル組を配列化し、pair chunkごとに最大値を縮約する。

このchunk化は計算量そのものを変えず、共分散と相関評価はいずれも概ね`O(n_ch^2)`で増加する。従って、9chでの実時間余裕は305chでの実時間性を保証しない。大規模アレイではactive方位だけを評価する設計と、別途305ch条件での時間・メモリ計測が必要である。

### 6.3 60秒連続観測

9ch、`fs=32768 Hz`、`n_fft=128`、159 beam/秒、全317方位、積分定数`T=10 s`の条件で60秒の連続観測を行った。信号は40度、1--4 kHzの帯域信号、`SL=0 dB re input RMS`、雑音は`NL=-32 dB re input RMS/sqrt(Hz)`とした。

| 項目 | 計測結果 |
|---|---:|
| scene生成 | 2.991 s / 60 s入力 |
| 共分散更新合計 | 0.196 s / 60 s入力 |
| 共分散更新中央値 | 3.21 ms / 1 s入力 |
| 共分散更新最大 | 4.96 ms / 1 s入力 |
| 最大相関評価合計 | 0.090 s / 60 s入力 |
| 最大相関評価中央値 | 1.47 ms / 1 s入力 |
| 最大相関評価最大 | 2.36 ms / 1 s入力 |

9chの評価条件では、共分散更新と最大相関評価の合計は60秒入力に対して約0.286秒であり、大きな処理ボトルネックは認められなかった。

各方位の初回更新直後は瞬時共分散がrank 1になるため、正規化した最大チャネル間相関はほぼ1になる。時間積分が進むと全周波数平均は2秒時点の約1.000から、10秒で0.838、20秒で0.760、40秒で0.748、60秒で0.747へ低下し、40秒付近から概ね安定した。60秒時点では1--4 kHz帯域の平均0.791に対し帯域外は0.738で、信号帯域の相関上昇は観測できる。一方、チャネル組の最大値は多数の組から最大を選ぶ統計的偏りを持つため、信号方位を鋭く分離する単独指標には不足する。次段階ではmedian、95 percentile、固有値による信号空間・雑音空間分離と併記して校正する。

### 6.4 64ch低周波広帯域観測

低周波向けの長開口条件として、64ch、受波器間隔0.5 m、開口長31.5 m、`fs=8192 Hz`、`NFFT=128`、`delta_f=64 Hz`を使用した。信号は40度、128--1024 Hzの帯域信号とした。等間隔ULAの`d <= lambda/2`に対応する基準周波数は1500 Hzであり、信号上限をこれより低くすることで、空間aliasの影響を避けて低周波広帯域の共分散形成を観測する。

60秒観測は完走し、scene生成4.87秒、共分散更新6.83秒、最大相関評価5.14秒であった。共分散更新と最大相関評価を合わせた解析実時間比は約0.200であり、この64ch条件でも逐次処理時間には余裕がある。一方、全317方位の共分散保持量は675,184,640 byte（約644 MiB）となった。チャンネル数をさらに増やす場合は、処理時間より先に常駐共分散のメモリ量が制約になる可能性が高い。

60秒時点の最大チャネル間相関は、信号帯域128--1024 Hzで0.932、1024--1500 Hzの比較帯域で0.872となり、信号帯域の相関上昇は確認できた。しかし、40度近傍の信号帯域平均は0.93168、信号方位から20度以上離れた方位では0.93173であり、方位選択性は確認できなかった。64chでは上三角だけでも2016チャネル組があり、その最大値は少数チャネル時より1へ偏りやすい。従って、`rho_max`は帯域観測値として保持するが、方式3の方位一致重みを単独で決める指標には採用しない。固有値による信号空間・雑音空間分離、チャネルペア相関のmedianおよび95 percentileを次の比較対象とする。

### 6.5 非対角相関統計と基線長別比較

積分時間10秒を変更せず、下三角の全2016チャネルpairについて、複素値を直接平均せず次式をpairごとに計算した。

```text
rho_ij(theta,k) = |R_ij(theta,k)| / sqrt(R_ii(theta,k) * R_jj(theta,k)), i > j
```

この`rho_ij`から最大、平均、中央値、95 percentileを方位・周波数ごとに求めた。また、等間隔ULAの基線index `abs(i-j)=1..63`ごとに平均し、`[n_direction,n_bin,n_baseline] = [317,65,63]`で保存した。共通計算は`spatial_correlation_statistics.py`、64ch固有の実験と成果物生成は`evaluations/beamforming/method3_low_frequency_64ch_correlation.py`へ分離した。

通常方位は2秒に1回だけ更新されるため、10秒積分に含まれる有効snapshot数は約5である。共有90度だけは約10であり、統計量間の比較条件を変えないため、今回の観測では積分時間を変更していない。

d_ij = norm(p_i - p_j)

### 6.7 異時刻snapshotのFFT後位相復元

方式3はchannelごとに異なる中心sampleからsnapshotを切り出すため、そのままのFFT spectrumは異なる絶対時刻を基準に持つ。これは方式3の設計に必要な位相復元が未実装の状態であった。各beamについてchannel中心の平均を共通基準時刻とし、実際の中心sample表から次を計算する。

```text
center_ref(b) = mean_ch(center(ch,b))
Delta_t(ch,b) = (center(ch,b) - center_ref(b)) / fs
X_restored(ch,k,b) = X_extracted(ch,k,b) * exp(-j 2 pi f_k Delta_t(ch,b))
R_inst(b,k) = X_restored(:,k,b) X_restored(:,k,b)^H
```

位相係数は中心表と同じくframe非依存なので、Accumulator初期化時に2 segment分の`[2,n_ch,n_bin,n_beam]`を1回だけ生成し、毎秒再利用する。解析tone試験では、8 sample異なる中心から切り出した2ch spectrumが補正前に異なる位相を持ち、補正後に一致すること、およびAccumulatorが生成する交差共分散が正の実軸へ揃うことを確認した。

同じ64chスパース条件を再評価した結果、`|R_ij|/sqrt(R_ii R_jj)`から作る最大、平均、中央値、95 percentile、および基線別統計は補正前と一致した。これは異常ではなく、位相復元が各channelへ単位絶対値の位相係数を掛けるため、各pairの`|R_ij|`が不変となるからである。従って、絶対値正規化相関だけでは時間軸復元の正否を評価できない。時間軸復元は方式3の共分散を正式に成立させるため必須だが、方位差を強めるには複素位相を保持する指標、steering整合量、または固有空間指標を別途評価する必要がある。

### 6.8 複素steering整合量と固有空間評価

時間軸復元後の共分散に対し、候補方位の物理到達遅延から複素steering `a(theta,k)`を生成し、次のsteering power fractionを計算した。

```text
C_steer(theta,k) = Re(a^H R(theta,k) a) / (trace(R(theta,k)) * a^H a)
```

rank-1信号空間がsteeringと一致すれば1、空間白色共分散なら`1/n_ch`となる。`a^H R a`を計算するまで`R_ij`の複素位相を保持するため、絶対値相関とは異なり方位整合を評価できる。

固有空間側はHermitian化した共分散を`R=U Lambda U^H`と分解し、主固有ベクトル`u1`について次を計算した。

```text
C_eig(theta,k) = |a^H u1|^2 / (a^H a)
principal_fraction(theta,k) = lambda1 / sum_i(lambda_i)
principal_to_noise(theta,k) = lambda1 / mean(lambda2,...,lambdaN)
```

EVDは方位8本ずつのchunkで実行し、全方位・全周波数の64x64固有ベクトルを同時保持しない。rank-1解析試験では整合steeringの`C_steer`と`C_eig`が1、直交steeringが0となり、空間白色共分散では`C_steer=principal_fraction=1/n_ch`となることを確認した。

64chスパース、60秒snapshot、128--1024 Hz帯域の結果は次のとおりであった。

| 指標 | 40度近傍 | 遠方 | 近傍 - 遠方 | 帯域平均peak方位 | 誤差 |
|---|---:|---:|---:|---:|---:|
| steering power fraction | 0.4630 | 0.0152 | 0.4478 | 40.44 deg | +0.44 deg |
| 主固有ベクトル整合度 | 0.7550 | 0.0154 | 0.7396 | 41.01 deg | +1.01 deg |
| 最大固有値占有率 | 0.5938 | 0.5608 | 0.0330 | 44.43 deg | +4.43 deg |
| 最大固有値/雑音固有値平均 | 99.03 | 88.83 | 10.20 | 43.86 deg | +3.86 deg |

複素steering整合量と主固有ベクトル整合度では、入力40度に対応する明瞭な周波数横断peakが現れた。従って、方式3の共分散は方位情報を保持しており、従来の絶対値相関が平坦だった主因は`|R_ij|`をpairごとに取って位相情報を捨てた評価方法にある。最大固有値占有率と固有値分離比は信号空間の強さを表すが、steeringを含まないため単独の方位推定指標にはしない。EVD処理時間は最終60秒snapshotに対して約9.08秒であった。

### 6.9 方位別共分散選択方式

#### 6.9.1 採否指標の責務

非対角正規化相関`|R_ij|/sqrt(R_ii R_jj)`は、信号帯域、基線長、pair構成など共分散の性質を説明する補助観測として保持する。ただし複素位相を失い、多数pairの最大値には極値biasもあるため、方位別共分散の採否Weightには使用しない。

採否の主指標は候補方位の複素steeringとの整合度とする。steeringをchannel normで正規化し、完成共分散に対する整合度を次で定義する。

```text
u(k,theta) = a(k,theta) / norm(a(k,theta))
eta(k,theta) = u(k,theta)^H R(k,theta) u(k,theta) / trace(R(k,theta))
```

`R`がHermitian半正定値なら`eta`は`[0,1]`に入る。空間白色雑音`R=sigma^2 I`では`eta=1/n_ch`、rank-1信号空間が`u`と一致すれば`eta=1`となる。

#### 6.9.2 瞬時powerの直接指数積分

完成した全方位共分散へ周期ごとに二次形式を適用せず、瞬時共分散`R_inst=X X^H`に対する次の恒等式を使用する。

```text
u^H (X X^H) u = |u^H X|^2
trace(X X^H) = norm(X)^2
```

FFT後に共通時間軸へ位相復元したsnapshot spectrumを`X[ch,bin,local_beam]`、今回のglobal方位indexを`direction_index[local_beam]`とする。事前計算steering tableは`A[ch,bin,direction]`、dtype`complex64`とし、アレイ位置、周波数軸、方位軸、音速が変わらない限り再生成しない。正規化table`U=A/norm_ch(A)`も同じshapeで保持する。

beam chunkごとに次を計算する。

```text
P_steering_inst[beam,bin] = |sum_ch(conj(U[ch,bin,direction_index[beam]]) * X[ch,bin,beam])|^2
P_total_inst[beam,bin] = sum_ch(|X[ch,bin,beam]|^2)
```

global積分状態のshapeとdtypeは次とする。

| 配列 | shape | dtype | 意味 |
|---|---|---|---|
| `steering_table` | `[n_ch,n_bin,n_direction]` | `complex64` | 物理steering |
| `normalized_steering_table` | `[n_ch,n_bin,n_direction]` | `complex64` | channel norm 1のsteering |
| `direction_covariance` | `[n_direction,n_ch,n_ch,n_bin]` | `complex64` | 方位別指数積分共分散 |
| `steering_power` | `[n_direction,n_bin]` | `float32` | `|u^H X|^2`の指数積分 |
| `total_power` | `[n_direction,n_bin]` | `float32` | `norm(X)^2`の指数積分 |
| `eta` | `[n_direction,n_bin]` | `float32` | 完成周期だけで生成する整合度 |
| `soft_weight` | `[n_direction,n_bin]` | `float32` | 次周期に用いるWeight |
| `weighted_covariance` | `[n_ch,n_ch,n_bin]` | `complex64` | 方位集約後の共分散 |

`direction_covariance`と同じ方位別忘却係数`alpha(theta)`を使う。

```text
P_steering_next = (1-alpha) P_steering_previous + alpha P_steering_inst
P_total_next = (1-alpha) P_total_previous + alpha P_total_inst
eta = P_steering / P_total
```

`P_total`が数値floor以下、非有限、`P_steering`が非有限、または比が許容丸め誤差を超えて`[0,1]`から外れるbinは`eta_valid=False`とする。無効etaを0という有効観測値へ変換せず、valid maskを別に保持する。同一snapshotと同一`alpha`を使う限り、直接積分した`eta`は完成共分散から計算する`u^H R u/trace(R)`と一致しなければならない。

Weight生成のため完成共分散へ全方位の二次形式を適用すると、演算量は次になる。

```text
O(n_beam * n_bin * n_ch^2)
```

一方、FFT直後に`|u^H X|^2`を計算して指数積分する追加演算は次である。

```text
O(n_beam * n_bin * n_ch)
```

64ch、1秒当たり159方位、65binでは、完成共分散への二次形式は概算`159*65*64^2 = 42,332,160`複素積和、FFT直後の直接射影は`159*65*64 = 661,440`複素積和となり、steering整合量の追加演算は概ね64分の1になる。この削減は近似ではなく、`u^H(XX^H)u=|u^H X|^2`による数学的同値変換である。

方位別共分散自体はMVDR、固有値評価、相関統計に必要なため従来どおり保持・積分する。ただしWeight生成だけを目的として、完成共分散へ`u^H R u`を再計算してはならない。完成共分散の二次形式は同値性を確認するtest/evaluationの参照計算に限定する。

実測では同一64ch入力を使い、steering整合量積分を無効にしたAccumulatorと有効にしたAccumulatorについて、1秒入力当たり中央値、最大処理時間、60秒入力に対する実時間比を比較する。一時メモリは少なくとも正規化steering table、`steering_power`、`total_power`、chunk内`projected`、2個の瞬時power配列のbyte数を記録し、完成共分散二次形式を全方位一括生成した場合の理論作業量とも比較する。理論上の64分の1削減が、実装上も過大なbroadcast配列を作らず得られていることを確認する。

#### 6.9.3 Weight生成

最初の実装・評価は二値化せず、周波数別threshold table`gamma_off[bin]`、`gamma_on[bin]`を受け取るsoft weightとする。

```text
g(theta,k) = clip((eta(theta,k)-gamma_off(k)) /
                  (gamma_on(k)-gamma_off(k)), 0, 1)
```

`gamma_off < gamma_on`を全binで必須とし、無効etaのweightは0とする。thresholdはruntime側で推定せず、evaluation側がnoise-onlyとtarget sceneから校正して渡す。従来値0.6は流用しない。noise-onlyの`eta`分布、理論値`1/n_ch`との差、target方位の分布、ROC、検出率、誤検出率から候補を決め、校正に使っていない方位・周波数・SNRで確認する。

二値Weightを将来有効化する場合は、前回完成Weightを状態として保持し、`eta>gamma_on`で0から1、`eta<gamma_off`で1から0へ遷移させる。中間領域は前回状態を保持する。softとbinaryは別の明示設定とし、戻り値型は変えない。

#### 6.9.4 完成周期と一周期遅延

方式3は1秒目に0--90度、2秒目に180--90度を更新するため、2秒を1完成周期とする。周期内の途中`eta`、途中Weight、片側方位だけの合成共分散は外部公開しない。

周期`t`完了時の順序を固定する。

1. 周期`t`のsteering power、total power、方位別共分散を完成させる。
2. 前周期で完成済みの`g_(t-1)`を周期`t`の完成方位別共分散へ適用し、公開候補`R_weighted,t`を作る。
3. 周期`t`の完成`eta_t`から次Weight`g_t`を生成し、double bufferの次側へpublishする。
4. 周期`t+1`が完成するまで、公開Weightは`g_t`、公開共分散は最後に完成した安全な共分散を保持する。

初回周期には`g_(t-1)`がないためweighted covarianceを生成せずfallbackする。同一区間の`eta_t`を同じ`R_t`の採否へ即時適用しないことで、偶然peakの自己強化を避ける。既存`DoubleBufferCallback`は「作業途中を公開しない」責務には合うが、本処理は2秒周期と方位別EMAを既に持つため、巨大なRuntime基底クラスは新設せず、専用結果型と小さなselector状態で同じpublish規則を実装する。

入力系列signatureは少なくともstream ID、アレイ位置、fs、NFFT、方位軸、音速を含む。signature変更、明示reset、shape例外、非有限入力例外では周期途中の更新count、未公開eta、次Weightを破棄し、前系列のWeight・保持共分散も失効させて初回周期から再開する。

#### 6.9.5 方位別共分散合成と成立条件

前周期Weightから周波数ごとに次を計算する。

```text
weight_sum(k) = sum_theta(g(theta,k))
R_weighted(k) = sum_theta(g(theta,k) R(theta,k)) / weight_sum(k)
N_eff_direction(k) = weight_sum(k)^2 / sum_theta(g(theta,k)^2)
```

次を全て満たすbinだけをweighted covarianceとして有効にする。

- `weight_sum`が設定`minimum_weight_sum`以上かつ有限
- `N_eff_direction`が設定`minimum_effective_direction_count`以上
- 使用する`R(theta,k)`、合成結果、traceが有限
- Hermitian相対誤差が設定上限以下
- Hermitian化後の最小固有値が`-psd_tolerance * max(trace, floor)`以上

合成後は`(R+R^H)/2`でHermitian化する。soft weightが非負で入力共分散が半正定値なら理論上PSDを保つが、有限精度と不正入力を検出するため固有値を確認する。

#### 6.9.6 fallback順序と保持期限

binごとに次の順序でfallbackする。結果には`source_code[bin]`、`valid_mask[bin]`、`reason_code[bin]`を必ず含める。

1. 成立条件を満たす方式3 weighted covariance
2. 呼出側から同周期に渡された方式2 covariance。有限、Hermitian、trace正、PSD tolerance内の場合だけ使用
3. 同一入力系列で過去に完成した方式3 weighted covariance。既定保持期限は2完成周期=4秒とし、設定可能にする
4. 現周期の有効な方位別共分散の対角powerを方位平均した対角共分散。空間相関を捨てるがHermitian PSDを保証する明示的safe fallback
5. いずれも成立しないbinはzero配列と`valid_mask=False`を返し、共分散として公開・使用しない

前回完成値は、保持期限超過、入力signature変更、reset、非有限化、Hermitian/PSD検査失敗で即時失効する。方式2がないことだけを理由に期限を延長しない。fallback理由は少なくとも`NO_PREVIOUS_WEIGHT`、`ZERO_WEIGHT_SUM`、`INSUFFICIENT_EFFECTIVE_DIRECTIONS`、`NONFINITE_WEIGHT`、`INVALID_WEIGHTED_COVARIANCE`、`METHOD2_USED`、`PREVIOUS_EXPIRED`、`DIAGONAL_FALLBACK_USED`、`NO_VALID_OUTPUT`を区別する。

#### 6.9.7 評価設計

実運用想定の中央密・外側疎64ch、開口32 m、最小間隔0.25 m、最大間隔1.0 mを使用し、座標と全基線分布を成果物へ保存する。最低限、次を独立sceneとして評価する。

1. noise-only
2. target-only
3. target+noise
4. 未校正方位を含む方位違いの単一低周波広帯域信号
5. 複数信号
6. 信号なしから信号ありへの遷移
7. 信号方位切替

### 6.11 方位別共分散そのものの方位選択性に関する仮説検証

Weight定義を確定する前に、「正しい候補方位では方位別共分散が強くなり、誤方位では弱くなる」という前提を独立に検証する。40度信号は120度候補snapshotにも含まれ得るため、`信号あり/なし`ではなく、`信号空間と候補steeringが整合/不整合`として解釈する。

候補処理が対角unitary変換`R_theta=D(theta) R D(theta)^H`だけなら、trace、固有値、Frobenius norm、`|R_ij|`は方位不変である。方式3で非unitary差が残るのは、候補ごとに異なる有限時間窓を切り出し、誤遅延が信号の自己相関時間に対して無視できない場合に限る。定常狭帯域tone、snapshotより十分長いcoherence time、または候補間遅延差が小さい短基線ではunitary近似へ近づく。広帯域、短い自己相関時間、長基線、大きい方位差では異なるsample集合によりcoherence低下が生じ得る。

現行中心表は`channel_center_samples[segment,ch,beam]`で各chのFFT窓中心を選ぶ。FFT前の時間移動後、`Delta_t=(center-center_ref)/fs`に対する`exp(-j2*pi*f*Delta_t)`で共通参照時刻へ戻す。従って完成`R(k,theta)`は候補方位へ整相済みの座標ではなく、共通参照位相の共分散である。etaは物理steering`u(k,theta)`を一度適用する定義とする。全ch同相ベクトルへの射影は、位相復元を行わず候補方位へ整相した共分散にだけ対応し、現行方式3へ使わない。

方位加算前の`R[direction,ch,ch,bin]`から、同一方位・周波数表示条件で次を保存する。

```text
P_trace = trace(R)
rho_ij = |R_ij| / sqrt(R_ii R_jj), i != j
eta = u^H R u / trace(R)
q_eig = lambda1 / trace(R)
g_eig = (lambda1-lambda2) / trace(R)
p = Re(u^H R u)
epsilon = norm_F(R-p*u*u^H) / norm_F(R)
```

`q_eig`と`g_eig`はrank-1集中度であり方位推定値ではない。unitary変換だけならtrace、`q_eig`、`g_eig`、相関絶対値は不変で、etaとsteering model残差だけが候補steeringとの複素整合を表す。`epsilon`は小さいほど候補rank-1 modelへ整合する。

64ch中央密・外側疎配置でnoise-only、狭帯域tone target-only、低周波広帯域target-only、広帯域target+白色雑音、広帯域target+空間相関雑音、複数信号を分離する。各sceneの方位、帯域、SL/NL、seed、座標、音速、NFFT=128、T=10 sを保存する。trace、相関mean/median/95/max、物理基線、d/lambda、pair構成、eta、q/g eig、epsilonについて正解値、±2度、20度以上遠方、margin、peak誤差、peak幅、正解外peak、grating候補、時間収束を記録する。

分類は`powerのみ`、`信号存在のみ`、`複素方位整合`、`snapshot不足支配`、`長基線/grating曖昧`とし、traceや絶対相関が正解方位で増えない場合は失敗として記録する。Weightの主指標、補助信頼度、noise threshold、soft mapping、fallback条件はこの分類後に再確定し、6.10のthresholdは校正候補のまま凍結する。

#### 6.11.1 決定論的確認と64ch評価結果

小規模な正定値共分散へ任意の対角unitary位相回転を加え、trace、全固有値、Frobenius norm、`|R_ij|`が数値精度内で不変になることをテストで固定した。現行方式3ではchannelごとの中心sample差をFFT後に`exp(-j2*pi*f*Delta_t)`で共通参照時刻へ戻すため、完成共分散へ適用するのは物理steeringであり、全ch同相vectorではない。直接指数積分etaと完成共分散から求めたetaの最大絶対差は6 scene全てで`4.2e-7`以下であった。従って、直接射影は近似ではなく同じsnapshotと忘却係数に対する数学的同値変換である。

分類結果は次のとおりである。

- trace: 共分散power。方位を区別せず、有限窓のpower揺らぎに支配される。
- 相関mean/median/95/max: coherenceと信号存在の補助説明。狭帯域では方位不変、広帯域でも主Weightには不十分。maximumは極値biasを持つ。
- 最大固有値比・gap: rank-1信号空間の補助信頼度。方位推定値ではなく、5 snapshot条件ではbiasが大きい。
- eta: 候補方位との複素位相整合を表す主指標。長基線の副peak・grating曖昧性はimagescと`d/lambda`統計で別途監視する。
- rank-1 steering model残差: etaと固有空間集中度を合わせた補助指標。主指標の代替ではなく整合モデルの説明に使う。

#### 6.11.2 Weight定義の確定

主指標はetaとする。traceや絶対相関値を直接Weightへ使わない。補助信頼度は、`total_power`が数値下限を超えること、etaが有限であること、完成共分散がHermitian/PSD条件を満たすこととする。固有値gapやrank-1残差は計算量と少数snapshot biasがあるため、常時Weight生成の必須条件にはせず診断量として保持する。

noise-onlyの周波数別eta分布から`gamma_off(k)`を誤検出率で校正し、target条件および未使用方位・周波数・SNRの検出率から`gamma_on(k)`を決める。soft weightは6.9.4のclip式を維持するが、6.10で得たq99/q10値は候補であり正式固定値ではない。Weight総和0、非有限、実効方位数不足、共分散品質不成立では6.9.6の順序で方式2へfallbackする。T=10 sは実装同値性の条件であり閾値確定にはsnapshot不足なので、次段階のROC校正はT=40 sとT=128 sも分離比較する。

#### 6.11.3 演算量と実測

64ch、159方位、65binでは完成共分散二次形式が42,332,160 complex MAC、FFT直後の直接射影が661,440 complex MACで64分の1である。固定配列を5回測定した中央値は4.74 ms対0.236 ms、最大は28.24 ms対0.319 msであった。比較用に完成共分散を明示生成すると338,657,280 byte、直接射影出力は82,680 byteである。実際の1秒入力処理については6.10の無効/有効比較どおり、中央値増加は約1.1 msである。共分散はMVDR、相関、固有値評価のため保持するが、Weight生成だけを理由に`u^H R u`を再計算しない。

成果物は`artifacts/beamforming/method3_covariance_directionality/`へ保存した。各sceneのNPZにはtrace、全pair相関統計、物理基線統計のpair数・範囲・代表値・標準偏差・IQR、`d/lambda`、pair構成、eta履歴、固有値指標、rank-1残差を含む。画像は全sceneで同じ方位軸、周波数軸、0--1色範囲を使用し、traceだけは全scene共通最大値で正規化した。
