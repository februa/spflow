# 共分散積分3方式比較設計

## 1. 目的と範囲

本検討はMVDR重みの性能比較には進まず、MVDRへ入力する周波数別空間共分散行列をどのように推定するかだけを比較する。

3方式は最終出力を共通して次のshapeとする。

```text
R[ch_i, ch_j, k]
shape = [n_ch, n_ch, n_bin]
dtype = complex64
```

比較時は同一の入力信号、窓関数、FFT正規化、周波数軸、積分時間を使用する。MVDR重み、BL改善量、採否判定は本設計の責務に含めない。

## 2. 共通記号

| 記号 | 意味 | 単位・shape |
|---|---|---|
| `fs` | サンプリング周波数 | Hz |
| `delta_f` | 共分散分析幅 | Hz |
| `N` | FFT長 `fs / delta_f` | sample |
| `X_m,q[k]` | channel `m`、snapshot `q`、bin `k` のFFT値 | complex |
| `R_q[k]` | snapshot `q` の瞬時空間共分散 | `[n_ch,n_ch]` |
| `tau[m,theta]` | 基準点に対する到来遅延 | s |
| `d[m,theta]` | `tau*fs`から作る切り出し遅延 | sample |

瞬時共分散は全方式で次式に統一する。

```text
R_q[i,j,k] = X_i,q[k] * conj(X_j,q[k])
```

窓のエネルギー利得とFFT長によるscaleは方式間で同一にする。相関係数ではscaleが相殺されるが、共分散の対角powerと方式間誤差を比較するため省略しない。

## 2.1 全方式共通の指数積分規約

3方式の積分はすべて指数積分とする。`coef`は過去共分散に掛ける係数ではなく、今回観測した瞬時共分散に掛ける更新係数として定義する。

```text
R_next = (1 - coef) * R_previous + coef * R_instantaneous
0 < coef <= 1
```

`coef`が小さいほど過去を長く保持する。時間方向に独立snapshotが`rate [snapshot/s]`で到来し、等価積分時間を`T [s]`とする場合は、次を基準式とする。

```text
coef = min(2 / (1 + rate*T), 1)
```

この式では`rate*T`が等価独立snapshot数である。指数重み`coef*(1-coef)^q`の有効snapshot数は次式となり、上の定義と一致する。

```text
N_effective = (2 - coef) / coef
```

初回更新ではゼロ行列からの立ち上がりbiasを避けるため、共分散の加重和`S`と重み和`Z`を別々に更新し、`R=S/Z`とする正規化指数積分も比較する。

## 3. 方式1: 1 Hz長時間FFT共分散

`delta_f=1 Hz`では`N=fs` sampleを1 snapshotとしてFFTする。1秒ごとに各binの外積を作り、時間方向へ積分する。

```text
x_frame: [n_ch, fs]
X:       [n_ch, n_bin]
R_inst:  [n_ch, n_ch, n_bin]
R_out:   [n_ch, n_ch, n_bin]
```

1 frame、1 binから得られる`R_inst`は外積1個なのでrank 1以下である。したがって「1秒に1回積分」は、1秒frameを複数時刻にわたって指数積分する意味とする。1秒分だけでフルランク共分散が得られるわけではない。

非重複1秒frameでは`rate=1 snapshot/s`とする。

### 長所

- 1 Hz分解能を直接得られる。
- 1秒区間にわたる狭帯域定常信号ではbin leakageを抑えやすい。

### 短所

- FFT次数、待ち時間、作業領域が大きい。
- 1秒内で変化する信号を1 snapshotへ混合する。
- 同じ積分時間ではsnapshot数が少なく、共分散rankと追従性が低い。

## 4. 方式2: 短FFT block共分散積分

分析幅を`delta_f Hz`とすると、FFT長は`N=fs/delta_f` sample、1秒あたりの非重複block数は`delta_f`となる。

```text
x_block: [n_ch, N]
X_block: [n_ch, n_bin]
R_block: [n_ch, n_ch, n_bin]
R_out:   [n_ch, n_ch, n_bin]
```

block開始位置は`q*N` sampleとする。overlapを使用する場合はsnapshot数ではなく、有効独立snapshot数も記録する。

非重複blockでは`rate=fs/N=delta_f snapshot/s`とする。overlapを使用する場合、単純なblock到来数を`rate`にすると相関snapshotを独立と数えてしまうため、窓とhopから推定した有効rateも併記する。

### 長所

- 方式1より低遅延・低FFT次数である。
- 同一積分時間でsnapshot数を増やせる。
- 現行`ShortFFTCovarianceAccumulator`がこの方式に相当する。

### 短所

- 到来時間差がblock内で無補償のため、有限窓内に入る波形区間がchannel間でずれる。
- 到来時間差が信号の相関時間やtransient長に対して無視できない広帯域・非定常信号では、空間相関を過小評価する可能性がある。

## 5. 方式3: 方位別遅延整合snapshot選別共分散

## 5.1 遅延表

到来遅延`tau[m,theta]`から、切り出しに使う整数遅延と位相復元に使う残差遅延を分離する。

```text
d_int[m,theta] = round(tau[m,theta] * fs)
tau_residual[m,theta] = tau[m,theta] - d_int[m,theta] / fs
```

高周波では1 sample未満の誤差でも相関値が変わるため、整数遅延だけで方式3を評価して完了とはしない。

## 5.2 中心サンプルテーブル

中心サンプルテーブルは`center[q,theta]`、shape `[n_snapshot,n_beam]`、dtype `int32`とする。

0–1秒では0–90度の159 beam、1–2秒では90–180度の159 beamを割り当て、同一方位内のblock中心が重複しないよう配置する。ただし、左右の方位半面を異なる時刻だけで評価すると、非定常sceneでは方位差と時間変化を分離できない。

そのため比較試験では次を分ける。

1. 定常scene: 指定どおり0–90度と90–180度を別秒へ配置し、基本式を確認する。
2. 非定常scene: frameごとに前後半の割当を交替し、各方位が同じ時間分布を持つようにする。

境界で0埋めしたsnapshotは相関値を歪めるため、中心テーブル生成時点で全channelの切り出し範囲がframe内に入る中心だけを有効とする。

## 5.3 方位別snapshot

beam `theta`、snapshot `q`、channel `m`について、次を切り出す。

```text
x_theta[q,m,n] = x[m, center[q,theta] + d_int[m,theta] + n]
shape = [n_snapshot,n_ch,N]
```

FFT後のshapeは`[n_snapshot,n_ch,n_bin]`である。

## 5.4 位相復元

式は実指数ではなく、虚数単位を含む複素位相回転でなければならない。

```text
phase_restore[m,k,theta] = exp(-j * 2*pi * f[k] * d_int[m,theta]/fs)
                              * exp(-j * 2*pi * f[k] * tau_residual[m,theta])
X_restored = X_shifted * phase_restore
```

符号は`x_shifted[n]=x[n+d]`のDFTが`exp(+j2*pi*f*d/fs)X(f)`になる規約に対する逆回転である。spflowの到来遅延・ステアリング符号と単体試験で一致させる。

重要なのは、位相復元しても有限窓が観測した時間区間の違いまでは元に戻らないことである。方式3の利点候補は「到来方向に対応する同一波面の有限時間区間を各channelから選ぶこと」であり、定常無限長信号では方式2と同値へ近づくべきである。

## 5.5 beam別共分散と相関値

位相復元後のsnapshotからbeam別共分散を時間方向へ指数積分する。この時間積分は方式1・2と同じ更新式を使い、各beamに対して独立な`S_theta`と`Z_theta`を保持する。

```text
R_theta[i,j,k,theta]
shape = [n_ch,n_ch,n_bin,n_beam]
```

正規化相関は次式とする。

```text
corr[i,j,k,theta] = abs(R[i,j,k,theta])
                      / sqrt(real(R[i,i,k,theta]) * real(R[j,j,k,theta]))
```

分母が0または数値下限未満の場合は相関0とする。対角を除く最大値を`rho_max[k,theta]`とする。

```text
rho_max[k,theta] = max(i != j) corr[i,j,k,theta]
w[k,theta] = rho_max[k,theta] > 0.6
```

`max`はchannel数が増えるほど偶然の高相関1組に反応しやすい。そのため0.6は採用済み閾値ではなく比較候補とし、同時に次も保存する。

- 非対角相関のmedian
- 非対角相関の95 percentile
- 最大固有値 / trace
- 閾値を超えたchannel pair比率

## 5.6 beam方向の指数積分

「同一方位方向の共分散を足す」は、各`theta`で時間snapshotを先に指数積分し、その後、同一binで有効beamをbeam方向へ指数積分する二段階と解釈する。

beam方向の更新係数を`coef_beam`とする。閾値選別を含むため、共分散加重和`S`と有効重み`Z`を別々に更新する。

```text
S_next[k] = (1-coef_beam)*S_previous[k]
              + coef_beam*w[k,theta]*R_theta[k,theta]
Z_next[k] = (1-coef_beam)*Z_previous[k]
              + coef_beam*w[k,theta]
R_out[k] = S_next[k] / Z_next[k]
```

`Z`が数値下限以下の場合は方式2の`R_out[k]`へfallbackする。集約後は数値誤差を除くため、`(R + R^H)/2`でHermitian化する。正定値性は保証されないため、固有値最小値とrankを観測するが、対角ローディングやMVDR重み評価は次工程とする。

## 5.7 `coef_beam`を先に式で確定しない理由

方式1・2の`rate`は時間当たりのsnapshot数として定義できる。一方、方式3のbeam方向積分では更新軸が時間ではなく角度であり、`rate*T`に対応する無次元量を一意に定義できない。

候補には少なくとも次がある。

- `rate_theta [beam/deg] * T_theta [deg]`
- 1回の走査で選択された有効beam数
- 相関のある隣接beamをまとめた有効独立beam数

また、指数積分は可換でないため、0度から180度へ走査する場合と逆順で`R_out`が変わる。これは時間EMAには自然な性質だが、角度集合の集約では方位方向の人工的な非対称性になる。

従って、`coef_beam`は算出式を先に固定せず、次の値を直接与えて共分散特性をシミュレーションする。

```text
coef_beam in {1.0, 0.5, 0.25, 0.125, 0.0625, 0.03125, 0.015625}
```

各係数で次の走査順を比較する。

- 方位昇順
- 方位降順
- broadsideから左右交互
- 固定seedのランダム順
- 昇順結果と降順結果の平均

係数ごとに`N_effective=(2-coef_beam)/coef_beam`、有効beam数、昇順・降順差、Frobenius誤差、trace、固有値、相関指標を保存する。走査順依存が許容範囲に収まり、広帯域・transient条件で方式2より真値誤差が改善する範囲を確認してから、`rate_theta`と`T_theta`の定義を確定する。

## 6. 現行実装との対応

| 対象 | 現状 |
|---|---|
| 方式1 | 専用実装なし |
| 方式2 | `ShortFFTCovarianceAccumulator`として実装済み |
| 方式3の遅延切り出し | `extract_delay_centered_snapshots`として一部実装済み |
| 方式3のbeam別共分散 | `DelayAlignedBeamCovarianceAccumulator`として一部実装済み |
| 中心サンプルテーブル | 未実装 |
| FFT後の位相復元 | 未実装 |
| `rho(k,theta)`と`w(k,theta)` | 未実装 |
| beam方向の正規化指数積分と方式2fallback | 未実装 |

現行`DelayAlignedBeamCovarianceAccumulator`は全beam共分散を単純加算しており、方式3の完成形ではない。

## 7. 比較評価条件

MVDRを使わず、共分散そのものを次のsceneで比較する。

1. spatially white noise only
2. 単一狭帯域tone
3. 単一band-limited broadband source
4. 複数狭帯域source
5. broadband source + white noise
6. 時間局在pulse / burst
7. 方位が前後半で変わる非定常scene

各sceneで次を記録する。

- Hermitian誤差
- traceとchannel別対角power
- rank、固有値、condition number
- 真値共分散または長時間参照共分散に対するFrobenius相対誤差
- source steeringに対する`a^H R a`
- `rho_max`、median、95 percentile、固有値coherence
- 有効beam数とfallback bin数
- `coef`と`N_effective`
- beam走査順を反転したときの共分散差
- 更新遅延、FFT回数、演算時間、保持メモリ量
- 定常sceneで方式2と方式3が同値へ近づくか
- broadband / transientで方式3が真値誤差を改善するか

## 8. 実装順序

1. 3方式共通の結果型と共分散品質指標を部品化する。
2. 方式1を参照実装として追加する。
3. 方式2を`delta_f`指定へ一般化する。
4. 中心サンプルテーブル生成を追加する。
5. 方式3の位相復元、beam別時間指数積分、相関指標を追加する。
6. `coef_beam`と走査順のシミュレーションを行う。
7. 方式3の閾値なし観測を行い、0.6を含む閾値sweepで誤検出・見逃しを評価する。
8. `coef_beam`の安定範囲から`rate_theta`と`T_theta`の定義を確定する。
9. 閾値を固定する前に、保持メモリ`n_ch^2*n_bin*n_beam`が実運用可能か確認する。

方式比較、parameter sweep、採否判定はexample/evaluation側に残し、共分散生成と品質指標だけを再利用部品とする。
