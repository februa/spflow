# 整相方式 設計結果

## 1. 目的と責務範囲

本書は、粗い分析幅の共分散から設計したMVDR重みを用いる整相方式について、整数遅延の寄与、方位別時間切り出し共分散との接続、運用上の成立条件をまとめる。

共分散の生成方式、方位選択性、積分schedule、完成状態の設計結果は`方位別時間切り出し共分散設計結果.md`を正とし、本書で再定義しない。本書は完成済み共分散を受け取った後の重み設計と整相出力を責務とする。

従来の「整数遅延+小数遅延固定整相出力から差分MVDR出力を引く」構成は比較対象として残すが、最初の主眼は差分補正枝の有無ではない。調査する問いは次の3つである。

1. 同一時間blockから作った粗い分析幅の共分散に対し、整数遅延を共分散推定とweight適用の前段に置くことで整相破綻を防げるか。
2. 方位別時間切り出しと共通時刻軸への復元により、整数遅延を信号経路に置かなくても、粗い分析幅の共分散から直接MVDRを成立させられるか。
3. どちらがtarget波形、干渉抑圧、連続性、計算量、latencyを含めて運用上現実的か。

## 2. 理想MVDRにおける整数遅延の等価性

周波数bin `k`の入力を`X[k]`、共分散を`R[k]`、target steeringを`a[k]`とする。channelごとの整数遅延`d_i`は、対角unitary行列

\[
D[k]=\operatorname{diag}\left(e^{-j2\pi f_k d_i/f_s}\right)
\]

で表される。遅延後入力、共分散、steeringは、同じ符号規約で

\[
X_D[k]=D[k]X[k],\qquad
R_D[k]=D[k]R[k]D[k]^H,\qquad
a_D[k]=D[k]a[k]
\]

と変換する。このときMVDR重みは

\[
w[k]=\frac{R[k]^{-1}a[k]}{a[k]^H R[k]^{-1}a[k]},\qquad
w_D[k]=D[k]w[k]
\]

であり、

\[
w_D[k]^H X_D[k]=w[k]^H X[k]
\]

が成り立つ。したがって、無限長、完全な時刻同期、同一の共分散標本、整合したsteeringという理想条件では、整数遅延の有無だけでMVDR出力は改善しない。この等価性を数値誤差レベルで確認することを最初の基準試験とする。

整数遅延の寄与は、この等価性が崩れる実装条件で評価する。具体的には、残留小数遅延の大きさ、有限長FIR近似誤差、FFT bin内の広帯域位相差、block境界の履歴不足、異なる時間区間から作った共分散、weight更新時の不連続、整数遅延bufferによるlatencyである。

現行の方位別時間切り出しは、channelごとに到来遅延した区間を取得した後、`exp(-j2πfΔt)`によって各spectrumをbeam内の共通中心時刻へ復元し、その後に共分散を作る。したがって、完成共分散のchannel位相座標は整数遅延適用後ではなく、通常入力と同じ共通時刻座標である。この共分散から設計したweightを整数遅延なし入力へ直接適用できるかが、今回の主検証仮説である。

一方、このweightを無変換のまま整数遅延後の信号へ適用するのは、weightと入力の座標不一致であり、整数遅延の効果検証にはならない。整数遅延後へ適用する場合は、`R_D=D R D^H`、`a_D=D a`として整数遅延座標へ変換してからweightを設計する。

## 3. 比較方式

主比較は、共分散の作り方と、weight適用時の整数遅延の2軸を混ぜずに構成する。

| ID | 共分散推定 | weight適用入力 | 拘束steering | 目的 |
|---|---|---|---|---|
| S0 | 同一時間block、整数遅延なし | 整数遅延なし | 物理遅延全体`a` | 粗い分析幅での旧来基準と破綻の再現 |
| S1 | 整数遅延後の同一時間block | 同じ整数遅延後信号 | 丸め後の残留steering`D a` | 整数遅延が粗い共分散の破綻を緩和できるか |
| T0 | 方位別時間切り出し+共通時刻復元 | 整数遅延なし | 物理遅延全体`a` | 主仮説。整数遅延なしの直接MVDR成立性 |
| T1 | T0の完成共分散を`D R D^H`へ座標変換 | 整数遅延後信号 | 変換後steering`D a` | T0との理想等価性とstreaming実装差 |

S1とT1で残留steeringを全channel共通の`1`に固定してはならない。整数sampleへの丸め後には最大0.5 sample相当の相対遅延が残るため、高周波ではこの残差が無視できない。`1`拘束は別の近似方式として扱い、主比較に混ぜない。

S1とT1も同一ではない。S1は整数遅延を適用した時系列から粗い共分散を改めて推定するため、同一時間blockであってもchannel間で観測する波面区間が近づく。T1はT0の完成共分散に対するunitary座標変換であり、共分散標本自体は変わらない。S0対S1は共分散推定に対する整数遅延の寄与、T0対T1はweight適用座標とstreaming実装に対する整数遅延の寄与を示す。

上記の主比較が成立した後に、「整数遅延+小数遅延固定整相」と「固定整相-差分MVDR」を運用比較として追加する。

## 4. 評価ケースと成分分解

各方式は、同じ入力波形、同じ観測時間、同じdiagonal loading規約、同じweight更新時刻で比較する。S0、S1、T0は推定する共分散自体が比較対象なので、共分散は共通化しない。T1だけはT0と同一標本のunitary座標変換とする。最初から多条件sweepを行わず、次の順序で方式と実装の正しさを分離する。

1. `target-only`、空間雑音なしでS0の破綻再現、S1とT0の成立性、T0とT1の等価性、歪みなし拘束、出力波形を確認する。
2. `noise-only`、channel無相関等分散雑音で、観測出力powerと`w^H R_n w`を照合する。
3. `interferer-only`、targetと同一周波数帯の独立干渉音に対する保護beamへの漏れ量を比較する。
4. `target+interferer+noise`で、成分別powerから混合出力が説明できることを確認する。
5. 低周波、中間周波、高周波、broadside、中間方位、endfireに展開する。

単一図だけで判定せず、target-only、noise-only、interferer-only、mixedのBLを同一方位軸、同一表示範囲、同一基準で保存する。BLは待受beam方位軸であることを明記し、一つの保護weightを固定して入力方位をsweepするbeam patternと混同しない。

## 5. 必須指標

`fixed_beam_single_source`と`slc_same_frequency_interference`の評価基準を併用し、次を必須とする。

- 歪みなし拘束`|w^H a|`と位相誤差。
- target-onlyの入力帯域積分RMSに対する出力level差`dB re input RMS`。
- target peak方位誤差、-3 dB幅、第一null幅、guard外peak。
- interferer-onlyの保護beam漏れlevelと固定整相からの低減量`dB re fixed beamformer output`。
- noise-onlyの実測power、`w^H R_n w`、white-noise gain。
- 出力波形のRMS、peak、NaN/inf、入力との相互相関、群遅延補正後の波形RMS誤差。
- 共分散のHermitian誤差、最小固有値、loading前後の条件数、weight norm、fallback率。
- block境界とweight更新境界のjump RMS、正常時jump RMSとの比。
- 1 block当たりの実行時間、実時間比、履歴buffer量、algorithmic latency。

T0とT1の理想等価性試験では、BLだけでなく、complex出力の最大絶対誤差とRMS誤差を直接比較する。S0対S1では共分散固有値、信号subspaceとsteeringの整合量、出力整相levelの改善を比較する。S1対T0では両方が成立するかだけでなく、残留小数遅延、波形誤差、latency、必要buffer量の差を比較する。運用上の採否閾値は、この数式同値性確認と代表条件の成分分解が完了した後に固定する。BLの複合scoreやparameter sweep選別には、視覚順位との一致度を未検証の指標を使用しない。

## 6. 検討順序と判定

実装と評価は次の順序で進める。

1. offlineの複素スペクトルでT0/T1等価性を確認し、符号、共役、正規化、axisを固定する。
2. 粗い分析幅の代表条件でS0の破綻を再現し、S1とT0がどちらも成立するかを比較する。
3. 方位別時間切り出し共分散をT0へ接続し、完成済み共分散から整数遅延なし入力用weightを作る。
4. streamingのS1とT0/T1を接続し、初回、終端、block境界、weight切替を評価する。
5. 固定整相と差分MVDRを加え、波形品質、干渉抑圧、計算量、latencyのtrade-offを比較する。
6. 代表条件で成立した後にだけ、周波数、方位、SNR、干渉方位差、積分時間、loadingのsweepへ展開する。

「整数遅延+MVDRが現実的」と判定するには、target保存とsame-frequency interferer抑圧だけでなく、streaming境界で未完成値を公開しないこと、fallback後も固定整相相当の安全側出力を維持すること、対象実行環境で実時間比が1未満であることを必須とする。
