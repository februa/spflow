はい。整理すると、あなたが言っている

> CBF では sidelobe に埋もれていた未検出信号が、ABF では sidelobe / interference floor が下がることで peak として現れる

という効果は、**「各方位を scan し、その方位を保護しながら他成分を抑える ABF / Capon / MVDR 型の空間スペクトル推定」**で起きる効果です。

一方、今の **A2 source-mask SLC** はそれとは違います。
A2 は「検出済み source を mask で保護し、その外側を non-source として下げる」方式なので、**未検出 source を peak として浮かび上がらせる方式ではありません**。むしろ mask 外にある未検出 source は抑圧対象になるリスクがあります。A2 の設計書でも、A2 は source-correlated leakage を下げる方式であり、未検出 source を non-source と見なすと削る可能性がある、と明記されています。

---

# 方式分類

## 方式を 3 グループに分けると分かりやすい

```text
Group 1: 探す方式
  未検出 source を peak として出したい方式。
  ABF / Capon / MUSIC / STFT-subband ABF はこちら。

Group 2: 残す方式
  source を消さず、固定整相の表示を安全に保つ方式。
  CBF / 固定整相 / shading / 遅延補正はこちら。

Group 3: 掃除する方式
  検出済み source 以外の leakage / sidelobe を下げる方式。
  A2 source-mask SLC はこちら。
```

社内説明では、この 3 分類を最初に出すと通じやすいです。

---

# 1. 固定整相 CBF / 小数遅延固定整相

## 処理フロー

```text
channel input
  ↓
周波数ごとの active channel / shading 選択
  ↓
整数遅延 + 小数遅延 FIR
  ↓
Delay-and-Sum
  ↓
BL / FRAZ / BTR
```

## 目的

```text
- 全 source を素直に表示する
- source を勝手に消さない
- 安定した baseline を作る
- A2 / ABF の fallback にする
```

## 未検出 source の peak は現れるか

**現れる可能性はあるが、ABF のように sidelobe floor を適応的に下げるわけではない**です。

つまり、

```text
強い未検出 source:
  CBF でも peak として出る

弱い未検出 source:
  既知 source の sidelobe や grating lobe に埋もれやすい
```

固定整相側は既に、運用アレイの周波数別 active aperture と Kaiser-Bessel shading により、200 Hz 以上で required peak margin 13 dB を満たす baseline になっています。

## 強み

```text
- source を勝手に消さない
- 物理的に説明しやすい
- 処理が軽い
- fallback として信頼できる
- source discovery の前段に向く
```

## 弱み

```text
- ABF のように干渉・sidelobe floor を適応的に下げない
- 強 source の sidelobe に弱 source が埋もれることがある
- non-source sector を選択的に下げられない
```

## 社内向け表現

> 固定整相は「探索用の安全な地図」。未知 source を消さないが、sidelobe が高いため弱い source は埋もれやすい。

---

# 2. B1: shading / active aperture / 小数遅延係数セット選択

## 処理フロー

```text
事前評価
  ↓
複数の shading / active aperture / FIR bank 候補を比較
  ↓
良い係数セットを選ぶ
  ↓
実行時は固定整相として適用
```

## 目的

```text
- CBF baseline の sidelobe envelope を下げる
- active aperture と shading の trade-off を調整する
- source を消さずに固定整相性能を底上げする
```

## 未検出 source の peak は現れるか

**少しは助ける可能性があります。**
ただし ABF ではありません。

```text
できる:
  sidelobe baseline を少し下げる
  未検出 source が CBF より見やすくなる可能性がある

できない:
  入力信号に応じて non-source だけを適応的に下げる
  ABF のように interference null を作る
```

今回の B1 横並び評価では、B1 は `hold=1253 / fail=3697` で、A2 より主候補として弱い結果でした。B2 は未評価です。

## 強み

```text
- source を消しにくい
- 固定整相 baseline の改善なので説明しやすい
- 実行時負荷はほぼ増えない
- 未知 source 探索との相性は良い
```

## 弱み

```text
- 改善幅は限定的
- 強く shading すると mainlobe が広がる
- ABF 的な adaptive suppression ではない
- 今回の単純 beta sweep では主候補になりにくい
```

## 社内向け表現

> B1 は「地図そのものを少しきれいにする方式」。未知 source を消さないが、ABF のような劇的な sidelobe 抑圧は期待しにくい。

---

# 3. B2: residual delay correction / 小数遅延補正

## 処理フロー

```text
calibration / 学習データ
  ↓
音速誤差・CH残差遅延・FIR近似誤差を推定
  ↓
遅延表または小数遅延選択を補正
  ↓
固定整相として適用
```

## 目的

```text
- 固定整相の位相ずれを減らす
- source peak の位置・形状を改善する
- model mismatch 由来の sidelobe / leakage を減らす
```

## 未検出 source の peak は現れるか

**固定整相の精度改善により、間接的に見えやすくなる可能性はあります。**
しかし、ABF ではありません。

```text
できる:
  位相誤差や音速誤差で崩れた peak を改善する
  model mismatch による sidelobe 増加を抑える

できない:
  入力中の強 source に合わせて他方位を適応的に抑圧する
  未検出 source を自動保護・自動強調する
```

B2 は設計上は「アレイ座標、音速、FIR 近似、実機応答の小さなずれを補正する方式」として整理されていますが、今回の横並び評価では補正係数 candidate 未定義のため未評価です。 

## 強み

```text
- 実機誤差がある場合に効く可能性がある
- source を消す方式ではない
- 実行時負荷が小さい
- 固定整相 baseline の品質改善になる
```

## 弱み

```text
- calibration が必要
- シミュレーションでは効果が出にくい可能性がある
- ABF 的な non-source 抑圧ではない
- 未評価なので8月主候補にはしにくい
```

## 社内向け表現

> B2 は「地図の座標補正」。方位推定や peak 形状の改善には効く可能性があるが、ABF のように背景を下げる方式ではない。

---

# 4. 従来 target-centric SLC

## 処理フロー

```text
固定整相 beam_output
  ↓
target beam を 1 本選ぶ
  ↓
guard 外 beam を reference にする
  ↓
target beam に漏れた interferer 成分を推定
  ↓
target beam から差し引く
```

## 目的

```text
- target beam に混入した特定 interferer leakage を下げる
```

## 未検出 source の peak は現れるか

**現れません。**
この方式は target beam の leakage を下げる方式であり、全方位の source discovery ではありません。

過去評価では、streaming 成分別 RMS では interferer 成分低減が見えた一方、final fixed-weight BL では interferer marker、guard 外 peak、first sidelobe が悪化しました。

## 強み

```text
- 特定 target beam の leakage 低減には使える条件がある
- 処理量が軽い
- target-centric な局所補正としては理解しやすい
```

## 弱み

```text
- 未検出 source を探さない
- BL 全体の sidelobe envelope を下げない
- 方位 off-grid に弱い
- marker だけ落ちて別方位が悪化しやすい
```

## 社内向け表現

> target-centric SLC は「1 本の target beam の漏れ込み掃除」。未知 source を浮かび上がらせる ABF ではない。

---

# 5. A2 source-mask SLC

## 処理フロー

```text
fixed_baseline beam_output
  ↓
source detection / known source input
  ↓
source mask 作成
  ↓
source mask 内 beam を reference
  ↓
non-source beam に出た source-correlated leakage を推定
  ↓
non-source beam から差し引く
  ↓
safety gate
  ↓
effective output
```

A2 の数式は、source reference `X_S` から non-source beam `d_b` を予測し、`y_b = d_b - eta h_b^H X_S` とするものです。source mask 内 beam は copy-through されます。

## 目的

```text
- 検出済み source は残す
- source mask 外の source-correlated leakage を下げる
- BL / FRAZ / BTR の non-source sector を ABF-like に掃除する
```

## 未検出 source の peak は現れるか

**基本的には現れません。**
むしろ、未検出 source が source mask 外なら、A2 はそれを non-source とみなす可能性があります。

```text
未検出 source が mask 外:
  抑圧されるリスクあり

未検出 source が potential source mask に入る:
  保護できる可能性あり

未検出 source が baseline でも見えない:
  A2 では保護できない
```

## 強み

```text
- 8月評価に間に合う実装量
- 処理量が軽い
- A2_safe は代表・負例で pass
- safety fallback を組み込みやすい
- 既知 source を残したまま non-source を下げられる
```

A2_safe / A2_aggressive は shortlist でどちらも full sweep `pass=330`、代表 4 条件 pass、負例 5 条件 pass、runtime p95 は約 0.2 でした。

## 弱み

```text
- 未検出 source discovery には向かない
- source mask 設計に強く依存する
- A2_aggressive は弱い未知 source を消しやすい
- 無相関雑音や source reference で説明できない sidelobe は下がらない
```

## 社内向け表現

> A2 は「検出済み source を残して、その他の leakage を掃除する方式」。探索ではなく、検出後の画面整理に近い。

---

# 6. A2_safe

## 目的

```text
- source 保護優先
- non-source を安全側に下げる
- 8月評価の主候補
```

## 特徴

```text
eta = 0.5
loading = 0.03
guard = wide
train_mode = one_block_delay
```

## 未検出 source の peak は現れるか

```text
単独では出ない。
ただし、A2_aggressive よりは消しにくい。
```

## 強み

```text
- 安全側
- source track を壊しにくい
- runtime に余裕あり
- fallback と組み合わせやすい
```

## 弱み

```text
- 抑圧量は A2_aggressive より小さい
- 未検出 source を自動発見する機能はない
```

## 社内向け表現

> A2_safe は「実運用候補」。未知 source 探索用ではないが、検出済み source の表示整理には最も安全。

---

# 7. A2_aggressive

## 目的

```text
- non-source 抑圧の上限を見る
- ABF-like 表示の比較候補
```

## 特徴

```text
eta = 1.0
loading = 0.1
guard = wide
train_mode = one_block_delay
```

## 未検出 source の peak は現れるか

```text
基本的には出ない。
むしろ mask 外なら消しやすい。
```

## 強み

```text
- 見た目は最も ABF-like
- non-source sector が大きく下がる
- source mask が正しければ強い
```

## 弱み

```text
- unknown / weak / detected miss source を消しやすい
- 標準運用には強すぎる
- source mask の品質に強く依存する
```

report でも、A2_safe は 8月評価の主候補、A2_aggressive は比較候補として扱い、負例で fallback 頻度や source 保護劣化が増えない場合に限定するとされています。

## 社内向け表現

> A2_aggressive は「見た目の改善上限」。source mask が正しい前提なら強いが、未知 source 探索には危険。

---

# 8. A2_safe_plus / potential source mask 付き A2

## 処理フロー

```text
fixed_baseline
  ↓
通常 source detection
  ↓
potential source detection
  - 弱い local peak
  - FRAZ ridge
  - BTR time-continuous ridge
  ↓
source mask = detected source ∪ potential source
  ↓
A2_safe
  ↓
candidate source suppression gate
  ↓
必要なら mask 拡張して再実行
  ↓
effective output
```

## 目的

```text
- A2 が未検出 source を勝手に消すリスクを下げる
- source と断定できない候補も保護側へ倒す
```

## 未検出 source の peak は現れるか

**完全ではないが、最も現実的に近いです。**

```text
baseline 上に弱い peak / ridge として見える:
  potential source として保護可能

baseline でも全く見えない:
  保護できない
```

## 強み

```text
- A2 の弱点を補える
- unknown source 保護側に倒せる
- discovery mode と cleanup mode の中間にできる
```

## 弱み

```text
- source mask が広がるため non-source 抑圧範囲が減る
- 誤検出 peak も保護してしまう
- ABF のように新しい peak を作るわけではない
```

## 社内向け表現

> A2_safe_plus は「消してはいけないかもしれない peak を保護する安全版」。ABF ではないが、A2 の未知 source 消去リスクを下げる。

---

# 9. MVDR / Capon / LCMV / GSC scan

## 処理フロー

```text
channel or beam-domain snapshot
  ↓
共分散推定
  ↓
scan 方位ごとに look direction を保護
  ↓
出力 power 最小化 / 制約付き null
  ↓
ABF / Capon spatial spectrum
```

## 目的

```text
- 各 scan 方位を順番に保護する
- その方位以外の成分を共分散に基づいて抑える
- sidelobe / interference floor を下げる
- 弱い source peak を浮かび上がらせる
```

## 未検出 source の peak は現れるか

**これが最も ABF らしくできます。**

未検出 source の方位を scan したとき、その方位が look direction として保護されます。
そのため、CBF では sidelobe に埋もれた source が、ABF / Capon map では peak として現れることがあります。

ただし、方式の使い方が重要です。

```text
scan ABF:
  未検出 source discovery に向く

target fixed ABF:
  target 以外を interference とみなし、未検出 source を消す可能性がある
```

過去の時間領域 MVDR / LCMV / GSC 検討では、制約点の null は作れたものの、BL 全体の guard 外 peak は改善せず、点 null では別方位に sidelobe が押し出される結果でした。

## 強み

```text
- 未検出 source を peak として出す ABF 効果に最も近い
- sidelobe / interference floor を下げられる可能性がある
- source discovery に使える
```

## 弱み

```text
- 計算量が大きい
- covariance 推定が必要
- coherent source や snapshot 不足に弱い
- 点 null 設計では sidelobe が別方位へ逃げる
- 8月評価の軽量本線には重い
```

リアルタイム側へ EVD / MUSIC / 全ビーム全周波数 solve を入れず、学習側とリアルタイム側を分離する方針は既に整理されています。

## 社内向け表現

> 本当に「CBF で埋もれた未知 source を浮かび上がらせる」なら、scan 型 ABF / Capon が本命。ただし、処理量・実装量・安定性の面で8月本線には重い。

---

# 10. STFT / subband beam-domain ABF / GSC

## 処理フロー

```text
fixed beam output[beam, sample]
  ↓
STFT / subband
  ↓
X[beam, freq, frame]
  ↓
freq ごとに guard / reference / covariance / weight
  ↓
frequency-selective ABF / GSC
  ↓
BL / FRAZ / BTR
```

## 目的

```text
- 周波数ごとに ABF-like suppression を行う
- 高域と低域で違う beam pattern を扱う
- 同一方位・異周波の成分も周波数軸で分ける
```

## 未検出 source の peak は現れるか

**できます。特に FRAZ 上の source discovery には相性が良いです。**

周波数ごとに処理できるため、

```text
6144 Hz の source
8192 Hz の source
10000 Hz の source
```

を別々に評価できます。

方式選定資料でも、STFT / subband beam-domain SLC は guard 幅、eta、loading、reference selection を周波数依存にでき、複数方位・複数周波数や同一方位・異周波に対応しやすい方式として整理されています。

## 強み

```text
- ABF-like source discovery に最も自然
- 周波数ごとに source peak を見られる
- 同一方位・異周波に強い
- 低域・高域の違いを扱いやすい
```

## 弱み

```text
- 実装量が増える
- 状態量が増える
- STFT / overlap / 再合成を考える必要がある
- 8月までの軽量リアルタイム本線には厳しい
```

## 社内向け表現

> STFT / subband ABF は「性能本命」。未知 source を周波数ごとに浮かび上がらせやすいが、8月評価には重い。

---

# 11. MUSIC / source discovery 補助

## 処理フロー

```text
fixed_baseline or STFT snapshot
  ↓
covariance
  ↓
MUSIC / peak tracking
  ↓
source 数・方位候補推定
  ↓
source mask 更新
  ↓
A2 or ABF へ渡す
```

## 目的

```text
- 未検出 source 候補を見つける
- A2 の source mask を補強する
- ABF / LCMV の制約方向候補を作る
```

## 未検出 source の peak は現れるか

**source discovery としてはできます。**
ただし、MUSIC は beam 出力波形そのものを改善する方式ではなく、**検出・学習側の補助**です。

方式選定資料では、MUSIC はリアルタイム出力に直接入れる処理ではなく、source 数、interferer 方位候補、制約方向候補を推定する学習・診断側の処理として位置づけられています。

## 強み

```text
- 未検出 source 候補を作れる
- A2 の mask 補強に使える
- 重い処理を学習側に寄せられる
```

## 弱み

```text
- 出力波形を直接改善しない
- covariance / EVD が必要
- coherent source や snapshot 不足に弱い
- 実装・評価項目が増える
```

## 社内向け表現

> MUSIC は「探すための補助センサ」。A2 の前段で source mask を増やす用途に向く。

# 社内説明用の推奨ストーリー

## スライド 1: 目的の違い

```text
CBF:
  すべての source を素直に見る。
  ただし sidelobe が高く、弱い source が埋もれる。

ABF:
  look 方位を保護し、他方位を抑える。
  CBF で埋もれた source が peak として見える可能性がある。

A2:
  source 検出後に、source 以外の leakage を掃除する。
  未検出 source を探す方式ではない。
```

## スライド 2: 今回の制約

```text
8月までに評価へ入る。
リアルタイム CPU 処理量を増やせない。
STFT / full ABF / channel-domain GSC は重い。
固定整相後の軽量処理を優先する。
```

## スライド 3: 候補比較

```text
fixed_baseline:
  探索用・fallback

A2_safe:
  8月評価主候補

A2_aggressive:
  比較候補

A2_safe_plus:
  未検出 source 保護を重視する追加候補

STFT / ABF:
  将来の性能本命
```

## スライド 4: A2 の限界

```text
A2 は source mask 外を non-source とみなす。
未検出 source は消す可能性がある。
そのため source detection 前には使わない。
```

## スライド 5: 未検出 source を扱う運用案

```text
Discovery mode:
  fixed_baseline または ABF/MUSIC 学習側で source 候補を探す。

Cleanup mode:
  source mask 確定後に A2_safe を適用する。

Presentation mode:
  A2_aggressive で表示改善の上限を見る。
```

## スライド 6: 推奨結論

```text
8月評価:
  fixed_baseline + A2_safe を主候補にする。

追加検討:
  A2_safe_plus で potential source mask を導入する。

将来:

  scan ABF / STFT-subband ABF / MUSIC 補助へ進む。
```

# 最終整理

あなたの理解は正しいです。

```text
CBFよりABFの方が sidelobe level が低く、
未検出 source peak が見えやすくなる
```

これは **scan 型 ABF / Capon / STFT-subband ABF / MUSIC 補助**で狙う効果です。

一方、今回の A2 は、

```text
検出済み source を残し、
source mask 外の leakage を下げる
```

方式なので、**未知 source を浮かび上がらせる方式ではありません**。

```text
  fixed_baseline

抑圧:
  A2_safe

```

と整理するのがよいです。
