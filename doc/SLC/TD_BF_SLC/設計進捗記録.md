## 6. 2026-07-04 beam-domain GSC 検討

### 6.1 目的

固定整相後のビーム出力から共分散を作り、beam-domain GSC として SLC を構成する方式を検討する。

### 6.2 試したこと

- 現行 narrowband SLC 診断の `projector` 処理を beam-domain GSC として再整理した。
- 固定整相後ビームの response matrix から target 応答ベクトルを作る方式を整理した。
- guard 外 reference beam に含まれる desired 成分を blocking projector で除去してから共分散を作る方式を定義した。
- 検討結果を `beam-domain_GSC検討.md` として追加した。

### 6.3 評価条件

今回は設計検討であり、新規評価はまだ実施していない。評価対象は、既存の小数遅延固定整相 + narrowband SLC 診断で使った以下の条件を出発点とする。

```text
前段固定整相:
  小数遅延固定整相

SLC 入力:
  固定整相後の beam output

共分散:
  blocking 後の guard 外 reference beam 出力から作成

重点周波数:
  6144 / 8192 / 10000 Hz
```

### 6.4 検討結果

beam-domain GSC の拡張候補は次とする。

```text
guard 外 reference beam
  + target response vector による blocking projector
  + blocking 後 reference covariance
  + loaded least-squares canceller
  + frequency-dependent guard / eta / loading
```

この方式は、固定整相後のビーム出力から共分散を作るため、現行の BL / FRAZ / BTR 評価と接続しやすい。また、時間領域 SLC で不足が出た場合に、STFT / サブバンド化した拡張方式へ接続できる。

### 6.5 問題点

- blocking は response matrix の精度に依存するため、小数遅延 FIR 応答込みの `steering_response()` を使う必要がある。
- reference beam 間の相関が高いと `R_uu` が悪条件になりやすい。
- 複数 target では、target 応答行列を使った multi-constraint blocking が必要になる。

### 6.6 次に試すこと

- narrowband 診断で blocking 有無を比較する。
- `R_uu` の condition number と `w` の norm を summary に追加する。
- `eta`, `loading`, `reference beam count` をスイープする。
- 時間領域 SLC L=1 の評価後、不足が明確になった場合に STFT / サブバンド beam-domain GSC の詳細設計に進む。

## 7. 2026-07-04 リアルタイム方式として時間領域 SLC を採用

### 7.1 背景

固定整相 + SLC をリアルタイム処理するため、1 秒分の入力を 1 秒以内に CPU で処理できる範囲を初期目標とする。

性能だけを考えると、SLC は周波数ごと、または帯域ごとに共分散を持つ方が有利である。ビームパターン、サイドローブ位置、小数遅延 FIR の残留誤差、干渉源スペクトルが周波数で変化するためである。

一方、周波数ビン別 SLC は `R_uu[f]`、`r_ud[f]`、`w[f]` を周波数ビン数分だけ持つ必要があり、各ビンの統計サンプル数も STFT フレーム数に制限される。現行の時間領域方式と比較すると、状態量、計算量、推定安定性のリスクが大きい。

### 7.2 決定

現行方式では、時間領域 SLC を採用する。

```text
beam_output: [n_beam, n_sample]
R_uu:        [n_ref, n_ref]
r_ud:        [n_target, n_ref]
w:           [n_target, n_ref]
```

共分散は周波数ごとには持たず、固定整相後の時間サンプルから 1 組だけ推定する。

このため、初期方式では guard も周波数ごとには切り替えない。固定整相 BL から作成した `guard(f)` 表は、単一の `guard_effective` を決める設計入力として使う。周波数ごとに guard を直接適用するのは、帯域別 SLC または周波数ビン別 SLC へ拡張した段階とする。

初期タップ長は `L=1` とする。これは広帯域に対して平均的なキャンセラであり、完全な周波数依存補償ではないが、実装が明確で、統計サンプル数を確保しやすく、CPU 処理量を管理しやすい。

### 7.3 拡張順

時間領域 SLC L=1 で不足が見えた場合、次の順で拡張する。

```text
1. 時間領域 SLC L=3〜8
2. 帯域別 SLC
3. 周波数ビン別 SLC
```

帯域別 SLC は、全ビン別より状態量が少なく、周波数依存も一定程度吸収できるため、周波数ビン別へ進む前の中間案とする。

### 7.4 設計書への反映

- `SLC基本設計書.md` に、現行方式が時間領域 SLC L=1 であることと、周波数依存 SLC への拡張順を追記した。
- `SLC方式選定検討.md` を、採用候補は時間領域 SLC、拡張候補は帯域別 / 周波数ビン別 beam-domain GSC となるよう更新した。
- `beam-domain_GSC検討.md` は、L=1 時間領域方式の本番経路ではなく、narrowband 診断の理論整理および将来拡張方式として位置づけ直した。

```text
HPBW_rad ~= 0.886 λ / D
D ~= 0.886 c / (f HPBW_rad)
```

`target_hpbw_deg = 8 deg`、`valid_frequency_hz_min = 256 Hz` とし、物理開口は `40.6 m` とした。

## 10. 2026-07-04 周波数別 Kaiser-Bessel シェーディング係数設計

### 10.1 目的

### 追記: L=1 時間領域方式の運用出力定義

```text
  出力パワー増加、出力パワー過大低下、推定キャンセル成分過大を検出
```

### 追記: 入力レベルに対する出力レベル・SN 改善評価

Beamforming の評価では、出力レベルが入力レベルに対して妥当かも確認する必要がある。特に、BL / FRAZ / BTR では表示上の peak 位置だけでなく、入力信号・入力雑音から期待される出力 SN と比較できる。

追加した評価基準:

```text
input_output_level_consistency
```

判定の基本式:

```text
target 信号:
  無歪み正規化されていれば、target 方向の出力 RMS は入力 target RMS と同程度

無相関雑音:
  sigma_out^2 = sigma_in^2 sum(|w_ch|^2)
  expected_spatial_snr_gain_db = -10log10(sum(|w_ch|^2))

矩形窓 delay-and-sum:
  w_ch = 1/N
  expected_spatial_snr_gain_db = 10log10(N)

channel shading 使用時:
  N_eff = (sum(g_ch))^2 / sum(g_ch^2)
  expected_spatial_snr_gain_db = 10log10(N_eff)
```

ここで `N` は active channel 数、`g_ch` は channel 方向の shading 係数である。スパースアレイでは周波数ごとに active subset が変わるため、SN 改善の期待値も周波数ごとに変わる。

BL / FRAZ のように時間方向または周波数方向へ平均した power を表示する場合、分析幅による雑音床低下も別に考える。独立平均数を `M` とみなせる条件では、分析由来の改善を `expected_analysis_snr_gain_db` として記録する。ただし、時間領域の瞬時波形出力では、明示的な平均、帯域制限、STFT 積分をしていない限り、この分析幅由来の改善を期待しない。

注意事項:

```text
- 信号 RMS レベルは dB20 で扱う。
- SN 改善量や雑音 power 比は dB10 相当の比として扱う。
- channel 間雑音が相関している場合、10log10(N_eff) の改善は成り立たない。
- coherent interferer は無相関雑音ではないため、SLC 評価の target/interferer 成分分離で別途見る。
- シェーディング、active channel、分析幅が周波数依存の場合は、周波数ごとに期待 SN 改善量を記録する。
```

この評価基準は、固定整相単一音源と SLC target-only では必須評価にした。複数音源、スパースアレイ設計、シェーディング設計、SLC 干渉条件では推奨評価として扱う。


### 補足: SN 改善量の 20log / 10log 表記

既存の BL / FRAZ / BTR は RMS 振幅レベルを `dB20` で表示している。そのため、表示上の信号レベル・雑音レベルは 20log で読む。

一方、無相関雑音の空間合成利得を理論式から導くと、雑音分散すなわち power 比として以下になる。

```text
sigma_out^2 = sigma_in^2 sum(|w_ch|^2)
SNR gain = -10log10(sum(|w_ch|^2))
```

これは dB20 表示と矛盾しない。RMS 振幅比として書けば、同じ改善量は以下である。

```text
SNR gain = 20log10(1 / sqrt(sum(|w_ch|^2)))
```

矩形窓の delay-and-sum では `sum(|w_ch|^2) = 1/N` なので、

```text
20log10(sqrt(N)) = 10log10(N)
```

となり数値は一致する。シェーディング時も同様に、

```text
20log10(sqrt(N_eff)) = 10log10(N_eff)
```

として評価する。したがって、BL / FRAZ / BTR の表示値は dB20 のまま扱い、SN 改善量を設計式で記述する場合だけ power 比の 10log 表記を併記する。


### 追記: dB 表記は単位ではなく基準量つき比率として扱う

工学的には dB は比率評価であり、単独では物理単位ではない。今後の BL / FRAZ / BTR / SLC 評価では、図軸、JSON key、設計書のいずれでも、絶対レベルなのか相対レベルなのか、何を基準にした dB なのかを明記する。

基本方針:

```text
絶対音圧 RMS:
  dB re 1 uPa RMS

振幅スペクトル密度:
  dB re uPa/sqrt(Hz)

power spectral density:
  dB re uPa^2/Hz
  または、設計で明確に定義した dB re uPa/Hz@ch

シミュレーション入力基準:
  dB re input RMS

BTR の時刻ごと正規化:
  dB re frame max

処理前後差分・margin・reduction:
  dB re before level
  dB re mainlobe peak
  dB re fixed beamformer output
```

評価基準カタログの各 criterion に `unit_reference` を追加した。これにより、metric が dB を含む場合でも、単なる `dB` ではなく基準量を併記する方針を設計書へ出力する。

また、BL/FRAZ/BTR の共通描画部品では既定ラベルを以下へ変更した。

```text
BL/FRAZ absolute-like level:
  RMS Level [dB re input RMS]

BTR relative level:
  Relative Level [dB re frame max]
```

実データ評価では、入力が音圧校正済みなら `dB re 1 uPa RMS`、周波数密度で評価するなら `dB re uPa/sqrt(Hz)` や `dB re uPa^2/Hz` を明示的に指定する。シミュレーションでは物理音圧校正がないため、`dB re input RMS` を既定の基準とする。

## 18. 2026-07-05 共分散の block-wise 忘却積分化

運用上は、1 秒全体を 1 回の共分散推定に使うよりも、固定長 block ごとに共分散を更新し、忘却係数で積分する方が現実的である。そのため、運用時間領域 SLC 診断を `slc_analysis_block_size` ごとの逐次処理へ変更した。

共分散更新は `BeamDomainSLC` 内部の `SlcCovarianceEstimator` を用い、各 block で次の式を適用する。

```text
R_k = alpha R_{k-1} + (1 - alpha) R_hat_k
r_k = alpha r_{k-1} + (1 - alpha) r_hat_k
alpha = exp(-block_time_sec / memory_time_sec)
```

成分別評価も、最後の係数を全区間へ適用するのではなく、mixed 条件で各 block に得た SLC 係数を target-only / interferer-only の同じ block へ適用する方式へ変更した。これにより、評価は実際の逐次運用に近い。

`slc_analysis_block_size = 8192`、`memory_time_sec = 1.0`、`fs = 32768 Hz` では `alpha = 0.7788` である。再評価結果は次の通り。

## 19. 2026-07-05 共分散積分時間の評価

`memory_time_sec` が短すぎる場合、各 block の `R_hat` に強く追従し、参照共分散 `R_uu` の条件数や SLC 重みが block 間で揺れやすくなる。そのため、`tap_len=1`、`slc_analysis_block_size = 8192`、`duration_s = 1.0`、`noise_level_db20 = -60 dB re input RMS` とし、target-only と異周波数 interferer 8192 Hz の条件で積分時間を評価した。

指数平均の独立 block 数の目安は、重み列 `(1-alpha) alpha^k` に対して `(1 + alpha) / (1 - alpha)` とした。`memory_time_sec < 0.5 s` では有効独立 block 数が 3 未満であり、異周波数 interferer 条件の `condition_number_relative_std` も 0.57 以上だった。これは低減量だけを見ると使えそうに見えても、共分散の値が十分に安定していない状態である。

```text
5.0              0.9512   40.008            30.194 dB re before   0.208              0.146
```

target-only 条件では、全ての積分時間で `raw_target_power_delta_db` は約 `-0.000006 dB re before level` 以内であり、target 自己消去は見られなかった。異周波数 interferer 低減は 1 秒が最も大きく、積分時間を長くするほど少し低下した。一方で、条件数の相対標準偏差は 1 秒から 5 秒で 0.242 から 0.208 へ小さくなり、長い積分ほど共分散値はやや安定する。

この結果から、共分散積分時間の採用範囲は `1.0 s` から `5.0 s` とする。既定は `1.0 s` とし、共分散安定性を優先する運用では `2.0 s` から `3.0 s` を候補にする。`5.0 s` は定常条件では使えるが、5 秒評価では 3 tau まで観測できないため、起動直後や方位変化がある条件では追従遅れに注意する。

### 19.2 積分時間の運用設定を 3 秒へ変更

運用設定は `memory_time_sec = 3.0 s` とする。1 秒は干渉低減量が最大だが、共分散値の安定性を重視して 2 秒から 3 秒側へ寄せる。3 秒条件では、target-only の `raw_target_power_delta_db = -0.000005 dB re before level`、異周波数 interferer 8192 Hz の `raw_interferer_reduction_db = 31.542 dB re before level`、`condition_number_relative_std = 0.234`、`realtime_factor = 0.151` であり、target 保護、干渉低減、共分散安定性、処理量のバランスが取れている。

### 19.3 SLC 後 BL 評価定義の修正

前回作成した SLC 後 BL は、各 beam をそれぞれ target beam として SLC を適用した結果を並べたものであり、保護 target beam を固定する SLC の評価として不適切だった。この定義では、interferer 方位の beam もその beam 自身を保護するため、BL 上で干渉が落ちないように見える。

修正後は、SLC の保護 target beam を `target_beam_index` に固定し、source 方位を走査した空間応答として BL を定義する。target 周波数では mainlobe 保護を確認し、interferer 周波数では target beam へ入る interferer 応答の低減を確認する。

3 秒設定の代表ケースでは、target 方位・target 周波数の変化は `0.000 dB re before level`、interferer 方位・8192 Hz 応答は `23.913 dB re before level` 低減した。これを今後の SLC BL 評価定義とする。

### 19.4 サイドローブ改善量評価への修正

BL 図の判定では、target mainlobe が悪化していないことは必要条件にすぎない。方式として価値があるかは、固定整相後の BL と SLC 後の BL を重ねたうえで、guard 外 sidelobe peak がどれだけ下がったか、また既知 interferer 方位の target beam 漏れ込みがどれだけ下がったかで判断する。

そのため、`protected_target_bl_summary` に次の指標を追加した。

```text
before_guard_outside_peak_db20
before_guard_outside_peak_azimuth_deg
after_guard_outside_peak_db20
after_guard_outside_peak_azimuth_deg
guard_outside_peak_delta_db
max_guard_outside_worsening_db
max_guard_outside_worsening_azimuth_deg
reduction_at_marker_db
```

`guard_outside_peak_delta_db` が 0 dB 付近または正の場合、BL 全体の sidelobe peak は改善していない。`reduction_at_marker_db` だけが大きい場合、その SLC は広い意味の sidelobe 低減方式ではなく、既知干渉方位に対する局所的な leakage canceller として扱う。方式選定では、target 保護、interferer marker 低減、guard 外 peak 改善、最大悪化量を同時に見て採否を決める。
