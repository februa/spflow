# 運用スパースアレイ設計

## 1. 目的

本書は、固定整相 + SLC で使用する運用スパースアレイを、設計スクリプトで作成し、実行時はファイルから読み込む方式として定義する。

アレイの CH 数、受波器位置、周波数ごとの active channel はコードに直書きしない。使用側は保存済み JSON を読み込み、`positions_m` の shape から CH 数を取得する。

---

## 2. 設計条件

今回の設計条件は次である。

```text
fs_hz                  = 32768 Hz
sound_speed_m_s        = 1500 m/s
maximum_frequency_hz   = 10000 Hz
valid_frequency_hz_min = 200 Hz
target_hpbw_deg        = 200 Hz 全開口から自動計算、約 1.27 deg
required_peak_margin   = 13 dB
evaluation_azimuths    = 60 / 90 / 120 deg
```

0 Hz は波長が無限大となるため、方位分解能や開口要求を定義できない。200 Hz 未満は全 CH を使い、200 Hz 以上は 200 Hz 全開口で決まるビーム幅を下限幅として active aperture と shading で概ね一定化する。

---

## 3. 受波器間隔の設計式

高域で等間隔アレイの空間 alias を避ける目安は次である。

\[
d \leq \frac{c}{2 f_{\max}}
\]

今回の条件では、

```text
c      = 1500 m/s
f_max  = 10000 Hz
d_max  = 1500 / (2 * 10000)
       = 0.075 m
```

このため、中心密配置では `0.05 m` 間隔を採用する。これは 10 kHz に対して `λ/2 = 0.075 m` より小さいため、高域 active 配置でグレーティングローブを避けやすい。

---

## 4. 開口長の設計式

broadside 近傍の ULA の半値幅目安として、次を使う。

\[
\mathrm{HPBW}_{rad}
\simeq
\frac{0.886 \lambda}{D}
\]

したがって、必要開口長は、

\[
D
\simeq
\frac{0.886 c}{f \mathrm{HPBW}_{rad}}
\]

である。

今回の設計では、ビーム幅の下限を `200 Hz` の物理全開口で決める。物理開口 `300.6 m`、音速 `1500 m/s` では、`HPBW ≈ 0.886 c / (f D)` より基準 HPBW は約 `1.27 deg` である。200 Hz 未満はこの幅を維持しようとせず、全 CH を使って低域の利得と長開口を確保する。

---

## 5. 物理配置

物理配置は中心対称 1 列アレイである。

positive 側の配置は次である。

```text
0.05  - 1.00  m : 0.05 m 間隔
1.20  - 4.80  m : 0.20 m 間隔
5.30  - 12.30 m : 0.50 m 間隔
13.30 - 30.30 m : 1.00 m 間隔
31.80 - 150.30 m : 1.50 m 間隔
```

これを 0 m を中心として左右対称に展開する。

```text
physical_n_ch = 305
aperture_m    = 300.6
min_spacing_m = 0.05
max_spacing_m = 1.50
```

中心部は高域用に密配置とし、外側は低域の長開口確保用に疎配置とする。

---

## 6. 周波数ごとの active channel

全周波数で全 CH を使うと、高域では外側疎配置の大間隔により grating lobe / 高 sidelobe が発生しやすい。

そのため、設計ファイルには周波数ごとの active channel index を保存する。

選定手順は次である。

1. `f < 200 Hz` では全 CH を active とする。`n_active = 305`、active aperture は `300.6 m` である。
2. `f >= 200 Hz` では `D(f) = D_ref f_ref / f` を基準に、200 Hz 全開口で決まる HPBW がそれ以上狭くならない active aperture を求める。
3. 候補として、開口内全 CH と、等間隔に近く間引いた sparse subset を作る。
4. exact-delay BL を `60 / 90 / 120 deg` で評価する。
5. 条件を満たす候補は active CH 数を優先し、全候補が未達の場合は margin 不足量が最小の候補を採用して後段 shading 評価へ渡す。

この方式により、200 Hz 未満では全 CH の長開口、200 Hz 以上では一定ビーム幅を狙う周波数別 active aperture を使う。

## 7. 保存ファイル
作成スクリプト:

```text

```

保存先:

```text
artifacts/beamforming/operational_sparse_array/operational_sparse_array_fs32768.json
artifacts/beamforming/operational_sparse_array/operational_sparse_array_design_table.csv
```

JSON には次を保存する。

```text
schema_version
fs_hz
sound_speed_m_s
valid_frequency_hz_min
maximum_frequency_hz
n_ch
aperture_m
positions_m: [n_ch, 3]
design_frequencies_hz: [n_frequency]
active_channel_indices_by_frequency: list[[n_active_ch]]
records
formula
```

使用側は `n_ch` を別途指定しない。`positions_m.shape[0]` を CH 数とする。

## 8. 読み込み方法

```python
from pathlib import Path

from spflow.beamforming import load_operational_sparse_array

array_definition = load_operational_sparse_array(
    Path("artifacts/beamforming/operational_sparse_array/operational_sparse_array_fs32768.json")
)

positions_m = array_definition.positions_m
n_ch = array_definition.n_ch
active_indices = array_definition.active_channel_indices_for_frequency(10000.0)
```

`active_channel_indices_for_frequency()` は、任意周波数に対して設計周波数表の ceiling 側を使う。これは、高い周波数で低周波用の広い active set を誤用するより安全側に倒すためである。

---

## 9. 生成結果

2026-07-05 時点の生成結果では、物理アレイは `305 ch`、物理開口は `300.6 m` である。
200 Hz 未満は全 CH を使う。200 Hz 以上は 200 Hz 全開口で決まる `target_hpbw_deg = 1.2666 deg` を基準に、周波数が上がるほど active aperture を縮める。

active aperture の代表値:

```text
10 Hz    : active 305 ch, aperture 300.6 m, all channels
128 Hz   : active 305 ch, aperture 300.6 m, all channels
200 Hz   : active  99 ch, aperture 300.6 m, beta  2.0, shaded worst margin 18.76 dB
256 Hz   : active 111 ch, aperture 261.6 m, beta 16.0, shaded worst margin 13.20 dB
512 Hz   : active  87 ch, aperture 114.6 m, beta  2.0, shaded worst margin 19.25 dB
1024 Hz  : active  85 ch, aperture  69.6 m, beta  5.0, shaded worst margin 14.55 dB
2048 Hz  : active 117 ch, aperture  32.6 m, beta  4.0, shaded worst margin 17.64 dB
4096 Hz  : active  63 ch, aperture  14.6 m, beta  2.0, shaded worst margin 18.20 dB
8192 Hz  : active  67 ch, aperture   7.2 m, beta  4.0, shaded worst margin 14.76 dB
10000 Hz : active  61 ch, aperture   6.0 m, beta  3.0, shaded worst margin 14.29 dB
```

注意点:

- `f < 200 Hz` は全 CH を使うが、200 Hz のビーム幅を維持する設計対象ではない。
- unshaded の sparse active 配置だけでは 13 dB margin を満たさない周波数がある。
- `f >= 200 Hz` の sidelobe margin は、active aperture と Kaiser-Bessel shading を組み合わせた結果として評価する。
- 200 Hz 以上の shading 後評価では、全評価周波数で `required_peak_margin_db = 13 dB` を満たした。

## 10. 小数遅延固定整相での性能評価

旧 125 ch / 40.6 m アレイの小数遅延固定整相評価結果は、新しい 305 ch / 300.6 m アレイには適用しない。
2026-07-05 の再評価では、運用アレイ JSON と shading JSON を同時に読み込み、整数遅延固定整相と小数遅延固定整相の両方へ同じ channel shading を掛けて BL margin を比較した。
この評価は Beamforming Evaluation の `fixed_beam_single_source` と `shading_design` に対応し、peak position、sidelobe margin、array file consistency、shading 後の有効 channel 数を確認する。

評価入力:

```text
array_json   = artifacts/beamforming/operational_sparse_array/operational_sparse_array_fs32768.json
shading_json = artifacts/beamforming/operational_shading/operational_kaiser_bessel_shading_fs32768.json
filter_bank  = fractional_delay_filter_bank_65x63.npz 相当
```

代表評価結果:

```text
256 Hz   : active 111 ch, aperture 261.60 m, beta 16.0, fractional worst margin 13.19 dB
512 Hz   : active  87 ch, aperture 114.60 m, beta  2.0, fractional worst margin 19.25 dB
1024 Hz  : active  85 ch, aperture  69.60 m, beta  6.0, fractional worst margin 16.34 dB
4096 Hz  : active  63 ch, aperture  14.60 m, beta  2.0, fractional worst margin 18.22 dB
10000 Hz : active  61 ch, aperture   6.00 m, beta  4.0, fractional worst margin 17.14 dB
```

このため、200 Hz 以上の代表評価点では、小数遅延 FIR の tap 長と遅延量子化を含めても、shading 後 BL の `required_peak_margin_db = 13 dB` を満たす。
ただし、この結果は BL margin 評価であり、FRAZ / BTR と SLC before / after の重ね合わせ評価は別途必要である。
