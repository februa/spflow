# 運用スパースアレイ設計

## 1. 目的

本書は、固定整相 + SLC で使用する運用スパースアレイを、設計スクリプトで作成し、実行時はファイルから読み込む方式として定義する。

アレイの CH 数、受波器位置、周波数ごとの active channel はコードに直書きしない。使用側は保存済み JSON を読み込み、`positions_m` の shape から CH 数を取得する。

---

## 2. 設計条件

今回の設計条件は次である。

```text
fs_hz                  = 32768 Hz
sound_speed_m_s        = 1500 m/s
maximum_frequency_hz   = 10000 Hz
valid_frequency_hz_min = 256 Hz
target_hpbw_deg        = 8 deg
required_peak_margin   = 13 dB
evaluation_azimuths    = 60 / 90 / 120 deg
```

0 Hz は波長が無限大となるため、方位分解能や開口要求を定義できない。したがって、0 Hz は設計ファイルの周波数表には含めるが、性能保証対象は `valid_frequency_hz_min` 以上とする。

---

## 3. 受波器間隔の設計式

高域で等間隔アレイの空間 alias を避ける目安は次である。

\[
d \leq \frac{c}{2 f_{\max}}
\]

今回の条件では、

```text
c      = 1500 m/s
f_max  = 10000 Hz
d_max  = 1500 / (2 * 10000)
       = 0.075 m
```

このため、中心密配置では `0.05 m` 間隔を採用する。これは 10 kHz に対して `λ/2 = 0.075 m` より小さいため、高域 active 配置でグレーティングローブを避けやすい。

---

## 4. 開口長の設計式

broadside 近傍の ULA の半値幅目安として、次を使う。

\[
\mathrm{HPBW}_{rad}
\simeq
\frac{0.886 \lambda}{D}
\]

したがって、必要開口長は、

\[
D
\simeq
\frac{0.886 c}{f \mathrm{HPBW}_{rad}}
\]

である。

今回の `target_hpbw_deg = 8 deg` では、性能保証下限 `256 Hz` 付近で約 `37.2 m` の開口が必要になる。安全率と物理配置の量子化を考慮し、物理開口は `40.6 m` とした。

---

## 5. 物理配置

物理配置は中心対称 1 列アレイである。

positive 側の配置は次である。

```text
0.05  - 1.00  m : 0.05 m 間隔
1.20  - 4.80  m : 0.20 m 間隔
5.30  - 12.30 m : 0.50 m 間隔
13.30 - 20.30 m : 1.00 m 間隔
```

これを 0 m を中心として左右対称に展開する。

```text
physical_n_ch = 125
aperture_m    = 40.6
min_spacing_m = 0.05
max_spacing_m = 1.00
```

中心部は高域用に密配置とし、外側は低域の長開口確保用に疎配置とする。

---

## 6. 周波数ごとの active channel

全周波数で全 CH を使うと、高域では外側疎配置の大間隔により grating lobe / 高 sidelobe が発生しやすい。

そのため、設計ファイルには周波数ごとの active channel index を保存する。

選定手順は次である。

1. 周波数ごとに開口式から必要開口を求める。
2. 候補として、開口内全 CH と、等間隔に近く間引いた sparse subset を作る。
3. exact-delay BL を `60 / 90 / 120 deg` で評価する。
4. mainlobe peak と mainlobe 外 peak の差が 13 dB 以上の候補を採用する。
5. 条件を満たす候補が複数ある場合は、active CH 数が少ない候補を優先する。

この方式により、低域では長開口 sparse subset、高域では中心密配置を使う。

---

## 7. 保存ファイル

作成スクリプト:

```text
examples/beamforming/design_operational_sparse_array_file.py
```

保存先:

```text
artifacts/beamforming/operational_sparse_array/operational_sparse_array_fs32768.json
artifacts/beamforming/operational_sparse_array/operational_sparse_array_design_table.csv
```

JSON には次を保存する。

```text
schema_version
fs_hz
sound_speed_m_s
valid_frequency_hz_min
maximum_frequency_hz
n_ch
aperture_m
positions_m: [n_ch, 3]
design_frequencies_hz: [n_frequency]
active_channel_indices_by_frequency: list[[n_active_ch]]
records
formula
```

使用側は `n_ch` を別途指定しない。`positions_m.shape[0]` を CH 数とする。

---

## 8. 読み込み方法

```python
from pathlib import Path

from spflow.beamforming import load_operational_sparse_array

array_definition = load_operational_sparse_array(
    Path("artifacts/beamforming/operational_sparse_array/operational_sparse_array_fs32768.json")
)

positions_m = array_definition.positions_m
n_ch = array_definition.n_ch
active_indices = array_definition.active_channel_indices_for_frequency(10000.0)
```

`active_channel_indices_for_frequency()` は、任意周波数に対して設計周波数表の ceiling 側を使う。これは、高い周波数で低周波用の広い active set を誤用するより安全側に倒すためである。

---

## 9. 生成結果

2026-07-04 時点の生成結果では、性能保証対象の全評価周波数で 13 dB 以上を満たした。

代表値:

```text
256 Hz   : active 17 ch, aperture 36.6 m, worst margin 13.03 dB
512 Hz   : active 17 ch, aperture 19.6 m, worst margin 13.66 dB
1024 Hz  : active 17 ch, aperture  9.6 m, worst margin 13.16 dB
2048 Hz  : active 19 ch, aperture  4.4 m, worst margin 13.04 dB
4096 Hz  : active 17 ch, aperture  2.4 m, worst margin 14.88 dB
10000 Hz : active 19 ch, aperture  0.9 m, worst margin 13.18 dB
```

注意点:

- この評価は exact-delay 幾何評価である。
- 実運用の固定整相では小数遅延 FIR を使用し、この幾何性能に近づける必要がある。
- 低域は少数 CH の長開口 sparse subset であるため、CH 数だけで性能を判断してはいけない。
- 高域は中心密配置を使うため、外側疎配置は active から外れる。

## 10. 小数遅延固定整相での性能評価

運用スパースアレイ JSON と保存済み小数遅延 FIR バンクを読み込み、周波数ごとの active channel subset に対して整数遅延固定整相と小数遅延固定整相を比較した。

使用した入力:

```text
array_definition:
  artifacts/beamforming/operational_sparse_array/operational_sparse_array_fs32768.json

fractional_delay_filter_bank:
  artifacts/beamforming/fractional_delay_filter_bank_65x63.npz
```

評価では、物理アレイ全体を常時使わない。各周波数で `active_channel_indices_for_frequency(f)` により active channel を取得し、その active 配置だけで遅延表と小数遅延 FIR 選択表を作る。これは、高域で外側の疎配置を誤って使うと grating lobe / 高 sidelobe を発生させるためである。

- 実運用で広帯域 FRAZ/BTR を一括評価する場合は、帯域分割後に active subset を切り替える処理として設計する。
