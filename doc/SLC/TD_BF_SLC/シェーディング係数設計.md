# シェーディング係数設計

## 1. 目的

固定整相時に channel 方向へ掛けるシェーディング係数を設計する。

シェーディング係数は、各 channel の寄与を制御する窓関数である。端 channel の係数を小さくすると、実効開口は短くなり、主ローブは広がる。一方で sidelobe は低下しやすくなる。

今回の目的は、運用スパースアレイの周波数別 active channel に対して、Kaiser-Bessel 窓を周波数ごとに設計し、後段のビーム補間が成立するように隣接待受ビームの -3 dB 主ローブ範囲を overlap させることである。

---

## 2. 入力

使用するアレイ定義:

```text
artifacts/beamforming/operational_sparse_array/operational_sparse_array_fs32768.json
```

この JSON から、物理 CH 数、センサ座標、周波数ごとの active channel index を読み込む。使用側は CH 数を手入力せず、ファイルから読み取る。

---

## 3. シェーディング式

active channel 数を `N`、channel index を `n = 0, 1, ..., N-1` とする。

Kaiser-Bessel 窓は次で与える。

```text
w[n] = I0(beta * sqrt(1 - x[n]^2)) / I0(beta)
x[n] = 2n / (N - 1) - 1
```

ここで `I0` は 0 次第 1 種変形 Bessel 関数、`beta` は窓形状パラメータである。

保存時は active channel 内の最大係数を 1 に正規化する。

```text
w_saved[n] = w[n] / max(w)
```

整相時の応答評価では、絶対利得が窓係数和に依存しないよう、次で正規化する。

```text
B(theta_signal | theta_wait)
  = sum_ch w[ch] exp(-j 2 pi f delta_tau[ch, theta_signal, theta_wait]) / sum_ch w[ch]
```

## 4. 保存形式

保存ファイル:

```text
artifacts/beamforming/operational_shading/operational_kaiser_bessel_shading_fs32768.json
artifacts/beamforming/operational_shading/operational_kaiser_bessel_shading_table.csv
artifacts/beamforming/operational_shading/operational_kaiser_bessel_shading_summary.png
```

JSON には full physical channel 長の係数を保存する。

active 外 channel の係数は 0 とする。これにより、将来の処理側では全 CH 入力に係数を掛けるだけで、周波数ごとの active subset を選択したのと同じ挙動にできる。

---

## 5. 評価条件

待受方位は、片舷アレイの設計に合わせて等 cos 空間で配置する。

評価は次の手順で行う。

1. ある待受方位 `theta_wait[i]` に固定整相ビームを張る。
2. 信号方位 `theta_signal` を 0 deg から 180 deg まで掃引する。
3. `B(theta_signal | theta_wait[i])` の peak を求める。
4. peak から 3 dB down までの連続した主ローブ範囲を求める。
5. 隣接待受ビーム `i` と `i+1` の 3 dB 主ローブ範囲が overlap するか確認する。

隣接評価値は次で定義する。

```text
overlap_margin[i]
  = right_3db_edge(theta_wait[i]) - left_3db_edge(theta_wait[i+1])
```

全ての隣接ペアで `overlap_margin[i] >= 0` なら、待受方位間のビーム補間が可能な条件を満たすと判断する。

あわせて、固定整相の sidelobe 条件として次も確認する。

```text
required_peak_margin_db >= 13 dB
```

---

## 6. 探索条件

探索した Kaiser-Bessel beta 候補:

```text
2, 3, 4, 5, 6, 8, 10, 12, 14, 16, 20
```

探索した待受方位数候補:

```text
151, 181, 241, 303
```

信号方位掃引点数:

```text
0 deg から 180 deg まで 3601 点
```

選択規則は、条件を満たす最小待受方位数を優先し、その中で最小 beta を選ぶ。これは、SLC の参照ビーム候補を確保しつつ、過剰なビーム数と過剰な開口縮小を避けるためである。

---

## 8. 注意点

- `beta=2.0` は矩形窓ではなく Kaiser-Bessel 窓であり、active 端係数は約 `0.439` である。
- 今回の選択規則では最小待受方位数を優先したため、`151` 本が選ばれた。
- SLC 評価などで `151` 本を固定したい場合、`candidate_n_beam_az_real` の下限を `151` にして再設計する。
- 保存係数は full channel 長であり、active 外は 0 である。
- 整相側では、窓係数を掛けたあと `sum(weights)` で正規化する必要がある。

## 9. 指定ビーム数で 3 dB 幅に近づける設計

SLC の参照ビーム数を先に決めたい場合は、待受方位数を探索せず、指定ビーム数を固定して beta だけを探索する。

追加スクリプト:

```text
examples/beamforming/design_operational_shading_for_beam_count.py
```

実行例:

```text
python examples/beamforming/design_operational_shading_for_beam_count.py --n-beam-az-real 151
```

保存先:

```text
artifacts/beamforming/operational_shading_fixed_beam/operational_kaiser_bessel_shading_151beam.json
artifacts/beamforming/operational_shading_fixed_beam/operational_kaiser_bessel_shading_151beam_table.csv
artifacts/beamforming/operational_shading_fixed_beam/operational_kaiser_bessel_shading_151beam_summary.png
```

この設計では、指定ビーム数のまま `minimum_three_db_overlap_margin_deg` が `target_overlap_margin_deg` に近くなる beta を選ぶ。既定値では `target_overlap_margin_deg = 0 deg` であり、隣接ビームの -3 dB 境界がちょうど接する状態を狙う。

注意点として、Kaiser-Bessel 窓は主ローブを広げることはできるが、矩形窓より狭くすることはできない。そのため、指定ビーム数が多くビーム間隔が既に 3 dB 幅より十分細かい場合は、`beta = 0` が最良となり、`meets_three_db_width_target = false` として保存する。

151 本指定の結果:

```text
selected_kaiser_beta = 0.0
selected_n_beam_az_real = 151
minimum overlap = 6.40 deg 以上
minimum 3dB width = 7.15 deg 以上
worst peak margin = 13.02 dB 以上
meets_three_db_width_target = false
```

これは、151 本では待受方位間隔が 3 dB 幅に対して十分細かく、channel shading だけでは 3 dB 幅をそこまで狭められないことを示す。ただし、beam-domain SLC の参照ビーム候補を確保するという目的には合っている。

## 10. BL sidelobe envelope 指標の追加評価

SLC 方式選定の見直しにより、シェーディング設計では `peak margin` だけでなく、BL 全体の sidelobe envelope と SNR loss も記録する。追加した指標は次である。

```text
effective_channel_count:
  N_eff = (sum(w))^2 / sum(w^2)

expected_spatial_snr_gain_db:
  10log10(N_eff)

snr_loss_vs_rectangular_db:
  10log10(n_active / N_eff)

worst_first_sidelobe_level_db_re_mainlobe_peak:
  評価方位ごとの第一副極のうち最も高い値。基準は mainlobe peak。

worst_sidelobe_95_percentile_db_re_mainlobe_peak:
worst_sidelobe_99_percentile_db_re_mainlobe_peak:
  guard 外 sidelobe 分布の 95% / 99% 点。基準は mainlobe peak。

worst_integrated_sidelobe_level_db_re_mainlobe_peak:
  guard 外 power 平均から求める integrated sidelobe level。基準は mainlobe peak。
```

2026-07-05 の再評価では、`artifacts/beamforming/operational_shading/operational_kaiser_bessel_shading_table.csv` に上記指標を追加した。代表的な worst-case は次である。

```text
min worst_peak_margin_db                         = 16.396 dB at 512 Hz
worst_first_sidelobe_level_db_re_mainlobe_peak   = -19.129 dB at 10000 Hz
worst_sidelobe_95_percentile_db_re_mainlobe_peak = -16.859 dB
worst_sidelobe_99_percentile_db_re_mainlobe_peak = -16.548 dB
worst_integrated_sidelobe_level_db_re_mainlobe_peak = -20.595 dB
max snr_loss_vs_rectangular_db                   = 0.261 dB at 768 Hz
```

この結果から、現行の `beta=2.0` Kaiser-Bessel shading は、全評価周波数で 13 dB peak margin を満たし、第一副極も少なくとも約 19 dB re mainlobe peak 低い。一方、SNR loss は最大でも約 0.26 dB であり、矩形窓に対する空間 SN 改善の損失は小さい。

したがって、固定整相側の改善 baseline としては、現行 `beta=2.0` の周波数別 active aperture + shading は妥当な baseline とする。次の検討では、`beta=0.0` 矩形 baseline、`beta=2.0`、より強い `beta` 候補を同じ `first_sidelobe / percentile / ISL / N_eff` 指標で比較し、SLC を使わずに BL envelope をどこまで下げられるかを Pareto curve として整理する。
