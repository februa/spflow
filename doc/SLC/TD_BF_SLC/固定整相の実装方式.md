# 固定整相の実装方式

## 1. 目的

本書は、spflow における時間領域固定整相処理の実装方式を定義する。

目的は、アレイ位置ベクトル、方向余弦データ、音速、サンプリング周波数をもとに、各受波器の到達時刻差を算出し、整数サンプル遅延と小数サンプル遅延 FIR に分解して固定ビーム出力を生成することである。

本方式では、待ち受け方位を等 cos 間隔で片舷ずつ作成し、各待ち受け方位に対して固定 Delay-and-Sum ビームフォーミングを行う。

---

## 2. 入出力の基本形

### 2.1 入力信号

入力信号はチャネルごとの時系列信号とする。

```text
x: [n_ch, n_sample]
```

### 2.2 アレイ位置ベクトル

アレイ位置ベクトルは、音響中心を原点とする。

```text
array_pos: [n_ch, 3]
```

各受波器位置を次のように表す。

\[
\mathbf{r}_m = [x_m, y_m, z_m]^T
\]

ここで、\(m\) はチャネル番号である。

### 2.3 方向余弦データ

待ち受け方位ごとの方向余弦ベクトルを次のように持つ。

```text
dir_cos: [n_beam, 3]
```

各ビーム方位の方向余弦を次のように表す。

\[
\mathbf{u}_b = [u_x, u_y, u_z]^T
\]

ここで、\(b\) はビーム番号である。

### 2.4 出力信号

固定整相後の基本出力は、チャネル平均後のビーム出力である。

```text
beam_output: [n_beam, n_sample]
```

必要に応じて、検証用にチャネル平均前の整相信号を保持してもよい。

```text
steered_channel_output: [n_beam, n_ch, n_sample]
```

ただし、通常の後段処理には `beam_output` を渡す。

---

## 3. 到達時刻差の符号規約

音響中心に対して、音の到達時間が早い場合を負、遅い場合を正と定義する。

したがって、受波器 \(m\)、ビーム \(b\) における到達時刻差は次で定義する。

\[
\tau^{arrival}_{m,b}
=
-\frac{\mathbf{r}_m^T \mathbf{u}_b}{c}
\]

ここで、\(c\) は音速である。

符号の意味は次の通りである。

```text
τ_arrival < 0 : 音響中心より早く到達
τ_arrival = 0 : 音響中心と同時に到達
τ_arrival > 0 : 音響中心より遅く到達
```

行列表現では、

\[
\mathbf{T}^{arrival}
=
-\frac{\mathbf{R}\mathbf{U}^T}{c}
\]

となる。

```python
arrival_delay_sec = -(array_pos @ dir_cos.T) / c
```

形状は次である。

```text
arrival_delay_sec: [n_ch, n_beam]
```

---

## 4. 整相補償遅延

整相では、早く到達したチャネルを遅らせ、遅く到達したチャネルに合わせる。

したがって、整相のために与える補償遅延は、到達時刻差の逆符号とする。

\[
\tau^{steer}_{m,b}
=
-\tau^{arrival}_{m,b}
\]

サンプル単位では、

\[
d^{steer}_{m,b}
=
f_s \tau^{steer}_{m,b}
\]

である。

```python
steering_delay_sample = -arrival_delay_sec * fs
```

ただし、時間領域処理では未来のサンプルを参照できないため、ビームごとに共通オフセットを加え、全チャネルの遅延を非負化する。

\[
d'_{m,b}
=
d^{steer}_{m,b}
-
\min_m d^{steer}_{m,b}
\]

```python
steering_delay_sample = steering_delay_sample - steering_delay_sample.min(axis=0, keepdims=True)
```

この共通オフセットは、ビーム内の相対遅延を変えないため、整相性能には影響しない。ただし、出力時刻には共通遅延が加わる。

---

## 5. 整数サンプル遅延と小数サンプル遅延への分解

補償遅延 \(d'_{m,b}\) を、整数サンプル遅延と小数サンプル遅延に分解する。

\[
n_{m,b} = \mathrm{round}(d'_{m,b})
\]

\[
\mu_{m,b} = d'_{m,b} - n_{m,b}
\]

ここで、\(n_{m,b}\) が整数遅延、\(\mu_{m,b}\) が小数遅延である。

```python
delay_int = np.round(steering_delay_sample).astype(int)
delay_frac = steering_delay_sample - delay_int
```

`round` 分解を使うことで、小数遅延はおおむね次の範囲に収まる。

```text
-0.5 sample ～ +0.5 sample
```

---

## 6. 小数遅延 FIR フィルタ

小数遅延は、事前設計済みの FIR フィルタ群から選択して適用する。

小数遅延量の候補を次のように定義する。

\[
\mu_q \in [-0.5, 0.5]
\]

例えば、\(N_{frac}\) 種類のフィルタを用意する場合、

\[
\mu_q = -0.5 + \frac{q}{N_{frac}-1}
\]

である。

```text
frac_grid: [n_frac_filter]
frac_filters: [n_frac_filter, n_tap]
```

各チャネル・各ビームの小数遅延 \(\mu_{m,b}\) に最も近いフィルタ番号を選ぶ。

```python
frac_filter_index = np.argmin(
    np.abs(delay_frac[..., None] - frac_grid[None, None, :]),
    axis=-1,
)
```

形状は次である。

```text
frac_filter_index: [n_ch, n_beam]
```

---

## 7. 小数遅延 FIR の基本設計式

小数遅延 \(\mu\) sample に対応する理想インパルス応答は sinc 関数で表せる。

FIR 長を \(N_{tap}\)、中心を

\[
M = \frac{N_{tap}-1}{2}
\]

とすると、窓掛け前のインパルス応答は、

\[
h_{\mu}[n]
=
\mathrm{sinc}(n - M - \mu)
\]

である。

実装では、窓関数を掛け、DC ゲインが 1 になるよう正規化する。

```python
def design_fractional_delay_filter(mu: float, n_tap: int) -> np.ndarray:
    n = np.arange(n_tap)
    center = (n_tap - 1) / 2

    h = np.sinc(n - center - mu)
    h *= np.hamming(n_tap)
    h /= np.sum(h)

    return h
```

この FIR は、実際には `center + mu` sample の群遅延を持つ。`center` は全チャネル共通の遅延として扱い、相対整相遅延には含めない。

---

## 8. 固定整相処理の流れ

ビーム \(b\) に対して、各チャネル \(m\) の信号に以下を適用する。

1. 整数サンプル遅延 \(n_{m,b}\)
2. 小数遅延 FIR \(h_{\mu_{m,b}}\)
3. チャネル平均

概念的には次である。

\[
x'_{m,b}[n]
=
(h_{\mu_{m,b}} * x_m[n-n_{m,b}])
\]

\[
y_b[n]
=
\frac{1}{N_{ch}}
\sum_{m=0}^{N_{ch}-1} x'_{m,b}[n]
\]

出力は、

```text
beam_output[b, n] = y_b[n]
```

である。

---

## 9. データ構造

固定整相に必要な設計データは次である。

```text
DelayTable
├─ delay_int           [n_ch, n_beam]
├─ delay_frac          [n_ch, n_beam]
├─ frac_filter_index   [n_ch, n_beam]
├─ frac_grid           [n_frac_filter]
└─ frac_filters        [n_frac_filter, n_tap]
```

実行時に最低限必要なのは次である。

```text
delay_int
frac_filter_index
frac_filters
```

`delay_frac` と `frac_grid` は、設計確認、ログ、検証に使用する。

---

## 10. 片舷処理と両舷結合

実用上は、右舷・左舷を片舷ずつ固定整相し、その後でビーム出力を結合する。

```text
right input
  ↓
right fixed beamformer
  ↓
beam_output_right [n_beam_right, n_sample]

left input
  ↓
left fixed beamformer
  ↓
beam_output_left [n_beam_left, n_sample]

concat
  ↓
beam_output_all [n_beam_all, n_sample]
```

片舷ごとの固定整相処理では、各舷の方向余弦テーブル、遅延テーブル、小数遅延 FIR 選択テーブルを独立に持ってよい。

後段処理が相対方位を要求する場合、結合後のビーム軸も相対方位として扱う。

---

## 11. 後段 SLC との関係

固定整相出力は、SLC の入力として次の形で渡す。

```text
beam_output: [n_beam, n_sample]
```

SLC では、target beam と reference beam を選び、target beam に混入したサイドローブ干渉を参照ビーム群から推定して差し引く。

固定整相部の遅延量や小数遅延 FIR は、SLC によって変更しない。

これにより、固定ビームフォーマのメインローブ形状を維持しながら、後段で適応的なサイドローブ抑圧を行う。

---

## 12. 設計方針まとめ

本方式の固定整相は、以下の方針で実装する。

```text
1. 音響中心を原点としたアレイ位置ベクトルを使う。
2. 到達時刻差は、早い場合を負、遅い場合を正とする。
3. 整相補償遅延は到達時刻差の逆符号とする。
4. 時間領域処理のため、ビームごとに補償遅延を非負化する。
5. 遅延量を round により整数部と ±0.5 sample の小数部に分解する。
6. 小数遅延は事前設計済み FIR フィルタ群から最近傍選択する。
7. 固定整相出力は [n_beam, n_sample] とする。
8. チャネル平均前の [n_beam, n_ch, n_sample] は検証用または高度化用とする。
9. 片舷ごとに固定整相し、必要に応じて後段で両舷ビームを結合する。
10. 後段の SLC は固定整相係数を変更せず、固定整相出力に対して適用する。
```
