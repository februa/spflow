# PRDFT FilterBank 実用設計方針

## 1. 目的

本書は、これまで進めてきた

- 均一帯域 PRDFT フィルタバンク
- 全帯域を高い周波数分解能で一様に扱う beamforming

という方針を、実用設計の観点から見直すための判断文書である。

既存の設計書は破棄しない。既存内容は

- 基準実装
- 比較対象
- 非均一化後の検証参照

として残す。

本書では、今後の主方針を

- 均一帯域一律設計から
- 帯域別の有効開口と帯域別の必要分解能を前提にした設計へ

切り替える。

---

## 2. 背景

既存設計では、全帯域を同一の周波数分解能で扱う前提を置いてきた。
これは基準系としては分かりやすく、比較や切り分けにも有効であった。

しかし実用上は、周波数帯域によって望ましいアレイ運用が異なる。

- 低域では広い開口を使いたい
- 高域では狭い開口を使いたい
- 高域では使用チャネル数を制限したい

この時点で、Overlap-save beamforming が必要とする有効な周波数分解能は
周波数ビンごとに同一ではない。

したがって、

- 全帯域を 1 Hz 分解能で同じように設計すること
- 全帯域で同じチャネル数・同じ共分散サイズを前提にすること

は、特に高域側で無駄が大きい。

---

## 3. 現在方式の実用上の問題

## 3.1 周波数分解能の過剰設計

高域では、低域ほど細かい周波数分解能を必要としない場合が多い。
例えば高域で実効的に `32 Hz` 程度の分解能で十分なら、
1 Hz ごとに

- 共分散行列を作る
- フィルタを設計する
- 重みを保持する

のは過剰である。

## 3.2 使用しないチャネルを含む計算の無駄

高域で使用チャネルを制限するなら、

- フィルタと信号の内積
- 共分散行列の計算
- MVDR 重み計算

の全てで、未使用チャネルを含めるのは無駄である。

特に共分散推定では、

- API 上は全チャネル次元を保ってもよい
- 実計算は使用チャネル部分行列だけで十分

である。

## 3.3 実装最適化余地を活かせていない

実用実装では、特に C++ 側では以下が候補になる。

- 使用チャネル部分だけの内積
- 使用チャネル部分だけの共分散推定
- Hermitian 性を使った上三角のみの計算
- 重み計算時だけ複素共役で復元

Python では効果が限定的でも、実運用系では無視しにくい。

---

## 4. 設計変更の前提

今後の beamforming 設計は、少なくとも以下を前提に置く。

1. 周波数ごとに使用チャネル集合 `used_channels[f]` が異なる。
2. 周波数ごとに必要な分解能 `delta_f[f]` が異なる。
3. 周波数ごとに有効開口と計算量を両立させる必要がある。
4. 一様な全帯域設計は基準系として残すが、主設計とはしない。

このため、今後の評価軸は

- 精度
- 処理量
- 実装複雑度
- 将来の非均一化への移行容易性

の 4 つとする。

---

## 5. 候補方式

## 5.1 方式A: 現行の均一帯域・均一分解能を維持する

内容:

- 均一帯域フィルタバンクを使う
- 全帯域を同じビン間隔で設計する
- 共分散や重みも全ビン独立に計算する

利点:

- 実装が単純
- 基準系として明快
- 数学的な比較がしやすい

欠点:

- 高域で分解能過剰
- 高域で未使用チャネル計算が残る
- 実装効率の伸びしろが小さい

位置付け:

- 今後は基準実装として残す
- 実用系の主方針にはしない

## 5.2 方式B: 均一帯域フィルタバンクを維持しつつ、周波数ビンをグルーピングする

内容:

- 解析・合成の基盤は当面、均一帯域フィルタバンクのまま使う
- beamforming 設計時だけ高域ビンをグルーピングする
- 代表ビンだけで `Rxx` と重みを計算し、近傍ビンへ共有する
- `used_channels[f]` を導入し、帯域ごとに使用チャネル数を変える

利点:

- 既存資産を活かしやすい
- 高域の無駄な 1 Hz 設計を減らせる
- 非均一化前でも計算量をかなり削れる
- 基準系との差分比較がしやすい

欠点:

- 真の非均一表現ではない
- グループ境界で近似誤差が出る
- ビン共有則の設計が必要

位置付け:

- 直近の主実装方針

## 5.3 方式C: 均一帯域フィルタバンクをやめ、非均一フィルタバンクまたはウェーブレットへ移行する

内容:

- 解析系そのものを非均一化する
- 低域は細かく、高域は粗く表現する
- 表現系と beamforming 分解能を最初から一致させる

利点:

- 設計思想として最も自然
- 高域の無駄を構造的に減らせる
- 将来的な実用系として最終形に近い

欠点:

- 実装難度が高い
- PR、遅延、帯域写像、Hermitian 整合を再検証する必要がある
- 既存検証資産をそのままは流用しにくい

位置付け:

- 中長期の本命候補

---

## 6. 採用方針

以下を採用する。

1. 既存の均一帯域・均一分解能設計は比較基準として残す。
2. 直近の実装方針は方式Bとする。
3. すなわち、当面は均一帯域フィルタバンクを土台に残しつつ、
   beamforming 設計と実装を帯域別分解能へ切り替える。
4. 将来の本命は方式Cとし、方式Bの API は非均一フィルタバンクへ移行しやすい形にする。

要するに、

- いきなり全面的に非均一化はしない
- しかし 1 Hz 全帯域一律設計も主方針から外す
- まずは「帯域別ビン群 + 帯域別使用チャネル」で実利を取る

という判断である。

---

## 7. 実装方針

## 7.1 帯域別使用チャネルの導入

各周波数ビンまたは各ビングループに対して、

- `used_channels`
- `channel_positions`
- `steering`
- `shading`

を切り替えられる構造を持つ。

beamforming の実計算は、常に使用チャネル部分だけで行う。

## 7.2 ビングループ単位の `Rxx` と重み計算

高域では、例えば

- `1 bin = 1 Hz`
- `32 bins = 32 Hz`

のように、複数ビンを 1 グループとして扱う。

各グループでは

- 代表ビンまたは代表周波数を定める
- その単位で `Rxx` を計算する
- その単位で重みを計算する
- 近傍ビンへ同じ重みまたは補間重みを使う

## 7.3 共分散計算の最適化

共分散推定は、

- 全チャネル正方行列を常に直接計算する方式

ではなく、

- 使用チャネル部分の小行列を直接計算する方式

を基本とする。

必要なら外部 API 境界でのみ

- 全チャネルサイズへ 0 埋め展開

を行う。

さらに実用実装では、Hermitian 性を利用して

- 上三角のみ計算
- 復元時に複素共役で下三角を補う

方式を検討対象とする。

## 7.4 内積計算の最適化

beamforming 時の

- フィルタと信号の内積

も、全チャネル固定ではなく

- 使用チャネル部分のみの積和

を基本とする。

## 7.5 API 設計の注意

将来の非均一フィルタバンク移行を見据え、
周波数軸を単純な一様 bin index だけで固定しない。

少なくとも論理的には、以下を分離する。

- 解析系が持つ bin
- beamforming 設計単位としての group
- group ごとの代表周波数
- group ごとの使用チャネル集合

---

## 8. 評価方針

方式Bと方式Cは、既存の均一帯域基準系に対して比較する。

主な比較項目は以下とする。

- 出力波形誤差
- サブバンド領域での重み適用誤差
- `Rxx` 推定誤差
- target 応答
- null 性能
- 処理量
- メモリ量

少なくとも以下の条件 sweep を行う。

- 低域
- 中域
- 高域
- bin center
- off-bin
- 使用チャネル数の少ない高域条件
- CBF と MVDR

---

## 9. 現時点での結論

実用上の主問題は、

- 全帯域一律の 1 Hz 設計が高域で過剰であること
- 高域で使用しないチャネルまで計算していること

の 2 点である。

したがって、今後の主設計方針は

- 均一帯域一律設計を基準系へ後退させる
- 実装系は帯域別分解能と帯域別使用チャネルへ移す

とする。

直近では

- 均一帯域フィルタバンクを残したまま
- 高域ビングルーピング
- 使用チャネル部分計算
- 小サイズ共分散

を導入する。

その上で、時間が許すなら最終的には

- 非均一フィルタバンク
- あるいはウェーブレット系

への移行を本命として進める。
